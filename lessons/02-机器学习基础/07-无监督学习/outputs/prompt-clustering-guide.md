# 聚类算法选择指南

聚类没有万能算法。正确的选择取决于数据规模、簇形状、噪声程度和业务目标。

## 决策清单

1. 你知道预期的簇数量吗？
   - 知道：K-Means 或 GMM
   - 不知道：DBSCAN（自动确定），或层次聚类（在树状图上切割）

2. 簇的形状是什么样的？
   - 大致球形（斑点状）：K-Means
   - 椭圆形且大小不同：GMM
   - 任意形状（月牙、环形、链状）：DBSCAN
   - 嵌套或层次结构：层次聚类

3. 数据包含异常值吗？
   - 包含：DBSCAN（自动标记噪声）或 GMM（低概率点）
   - 不包含：K-Means 即可

4. 需要软分配（概率）吗？
   - 需要：GMM 给出每个点属于各簇的概率
   - 不需要：K-Means 或 DBSCAN 输出硬分配

5. 数据集有多大？
   - 1 万以内：任何算法都可以
   - 1 万 - 100 万：K-Means（快），Mini-Batch K-Means（更快）
   - 100 万以上：Mini-Batch K-Means 或 BIRCH。层次聚类太慢

## 各方法的适用场景

**K-Means**：默认的起点。速度快（O(n·K·迭代)），简单，适用于许多问题。使用肘部法或轮廓系数选择 K。局限：假设球形簇，对初始化敏感（使用 K-Means++ 或多次运行），不能很好地处理不同大小的簇。

**DBSCAN**：最适合发现任意形状的簇和自动识别异常值。两个参数：eps（邻域半径）和 min_samples（最小密度）。不需要指定 K。局限：当簇密度差异大时效果差，调整 eps 可能需要技巧（使用 k-距离图）。

**层次聚类（凝聚法）**：构建合并树。适合在多个粒度层次上探索簇结构（在树状图不同高度切割）。Ward 连接方式最适合紧凑的簇。单连接能发现细长簇但对噪声敏感。局限：O(n²) 内存和 O(n³) 时间，对大数据集不实用。

**GMM（高斯混合模型）**：软聚类，使用概率分配。将每个簇建模为具有自己的均值和协方差的高斯分布。当簇是椭圆形或重叠时比 K-Means 更好。使用 BIC（贝叶斯信息准则）选择成分数量。局限：假设高斯分布，对非凸形状可能失败，对初始化敏感。

## 评估聚类质量（无标签）

| 指标 | 衡量内容 | 取值范围 | 使用场景 |
|------|---------|---------|---------|
| 轮廓系数 | 内聚度 vs 分离度 | -1 到 1（越高越好） | 比较 K 值或算法 |
| 惯性（簇内平方和） | 簇的紧密程度 | 0 到 ∞（越低越好） | K-Means 肘部法 |
| BIC / AIC | 模型拟合 + 复杂度惩罚 | 越低越好 | 选择 GMM 成分数量 |
| Calinski-Harabasz 指数 | 簇间/簇内方差比 | 越高越好 | 快速比较 |
| Davies-Bouldin 指数 | 簇间平均相似度 | 越低越好 | 惩罚重叠簇 |

## 常见错误

- 不标准化特征直接运行 K-Means（尺度大的特征会主导距离计算）
- 在 2D 可视化中目测选择 K，而实际数据是高维的（使用轮廓系数）
- 对非球形簇使用 K-Means（月牙形或环形数据需要 DBSCAN）
- DBSCAN 的 eps 设得太大（所有点在一个簇）或太小（所有点都是噪声）
- 将聚类标签当作真实标签（聚类是探索性的，需要领域知识验证）
- 对超过 2 万点的数据运行层次聚类（内存和时间爆炸）
