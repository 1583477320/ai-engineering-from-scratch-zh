# 分类基线指南

在尝试复杂模型之前，先用逻辑回归建立基线。它的训练时间在秒级，输出概率，且完全可解释。大量真实问题不需要比这更复杂的模型。

## 决策清单

### 1. 决策边界是否可能是线性的？

- **是**：逻辑回归很可能就够用了
- **否**：仍然需要用它作为基线，衡量后续复杂模型的提升幅度

### 2. 特征维度是多少？

- **< 50**：标准逻辑回归即可
- **50 ~ 10,000**：加上 L2 正则化（Ridge）
- **> 10,000**（如文本 TF-IDF 特征）：使用 L1 正则化（Lasso）或 `SGDClassifier`

### 3. 数据集是否不平衡？

- **比例 < 5:1**：无需特殊处理
- **5:1 ~ 50:1**：使用 `class_weight='balanced'`
- **> 50:1**：结合类别权重 + 合适评估指标（精确率/召回率/F1）

### 4. 特征尺度是否一致？

- **必须在训练前标准化**。逻辑回归使用梯度下降，特征尺度不一致会严重拖慢收敛。

### 5. 是否有缺失值？

- 逻辑回归不能处理缺失值，必须先填充。
- 数值特征用中位数填充，类别特征用众数填充。

## 何时逻辑回归足够

- 二分类问题，特征与目标间主要是线性关系
- 需要概率输出（不仅仅是类别标签）
- 需要模型可解释性（权重系数表示特征重要性的方向和大小）
- 训练数据量较小（几百到几千样本）
- 需要实时预测（推理就是一次点积运算）
- 监管或合规要求模型可解释

## 何时需要升级模型

- 准确率远低于目，且特征工程已无提升空间
- 特征与目标的关系明显非线性（检查残差图）
- 有大量表格数据（10K+ 行）：尝试梯度提升（XGBoost 或 LightGBM）
- 特征间存在复杂交互，多项式特征也无法捕获
- 输入是图像、文本或序列数据：逻辑回归在原始输入上不适用

## 预处理步骤

1. **先划分训练集/测试集**，再做任何预处理（防止数据泄露）
2. **处理缺失值**：数值用中位数，类别用众数
3. **编码类别变量**：低基数（< 10 值）用 one-hot，高基数用目标编码
4. **标准化数值特征**：`StandardScaler`，只在训练集上 fit
5. **训练逻辑回归**：`C=1.0`（默认正则化强度）
6. **评估**：混淆矩阵 + 精确率 + 召回率 + F1，不仅仅是准确率
7. **调整阈值**：默认 0.5 通常不是最优值，在验证集上搜索 0.1 到 0.9

## 常见错误

- 在不平衡数据上只看准确率（模型全猜多数类也能高分，但毫无用处）
- 忘记标准化特征（收敛慢，解的质量差）
- 用测试集调决策阈值（用验证集或交叉验证）
- 不做基线直接上 XGBoost（失去可解释性，且没有参照点）
- 忽略多重共线性（高度相关的特征会增大权重系数的方差）

## 快速参考

| 场景 | 模型 | 正则化 | 关键参数 |
|---|---|---|---|
| 少量特征，需要可解释 | LogisticRegression | L2（默认） | `C=1.0` |
| 多特征，部分不相关 | LogisticRegression | L1 | `penalty='l1', solver='saga'` |
| 高维稀疏（文本） | SGDClassifier | L1 或 ElasticNet | `loss='log_loss'` |
| 类别不平衡 | LogisticRegression | L2 | `class_weight='balanced'` |
| 需要概率输出 | LogisticRegression | L2 | `predict_proba()` |
| 只需类别标签 | LinearSVC | L2 | 大数据集比 LR 更快 |
