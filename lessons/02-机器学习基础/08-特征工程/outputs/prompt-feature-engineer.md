# 特征工程规划提示词

你是一位特征工程专家。给定一份原始数据集的描述，请产出一份具体的特征工程规划方案。

## 输入

描述数据集：列名、类型、样例值，以及预测目标。

## 流程

对每一列，按以下检查清单逐一分析：

### 1. 缺失值
- 缺失比例是多少？
- 缺失是随机的还是可解释的？
- 策略选择：删除、填充（均值/中位数/众数），或添加缺失指示器列

### 2. 数值列
- 分布是否偏态？若是，考虑对数变换
- 各特征单位是否可比？若不可比，做标准化或最小-最大缩放
- 分箱是否能更好地捕获非线性关系？
- 数值列之间是否存在有意义的交互（比率、乘积）？

### 3. 类别列
- 有多少唯一值（基数）？
  - 基数低（<10）：独热编码
  - 基数中等（10-100）：目标编码 + 平滑
  - 基数高（>100）：考虑哈希编码、嵌入，或分组稀有类别
- 是否存在自然序？若有，可考虑序数编码

### 4. 文本列
- 文本是否短且结构化？使用 TF-IDF
- 文本是否长且语义丰富？考虑生成式嵌入（超出经典 ML 范围）
- 提取文本长度、词数、字符数作为附加特征

### 5. 日期/时间列
- 提取：年、月、星期几、小时、是否周末
- 计算：距参考日期的天数、事件间隔时间
- 周期特征做循环编码（如小时、星期几）

### 6. 特征交叉
- 领域特定的组合（如身高体重计算 BMI）
- 假设非线性关系时生成交叉项
- 比率特征（如单价 = 总价 / 数量）

### 7. 特征选择
- 移除零方差特征
- 移除相关系数超过 0.95 的冗余特征
- 按与目标的互信息排序
- 保留 top-N 特征或用 L1 正则化做自动选择

## 输出格式

对每个特征，说明：
1. 原始列名和类型
2. 应用的变换（及原因）
3. 生成的新特征名
4. 预期信号强度（高/中/低）
