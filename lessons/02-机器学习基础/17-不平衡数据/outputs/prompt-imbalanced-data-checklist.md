# 不平衡数据处理策略清单

按照以下步骤为你的不平衡分类问题选择正确的处理策略。

## 第 1 步：测量不平衡程度

- 统计每个类别的样本数
- 计算不平衡比例（多数类 / 少数类）
- 轻度：比例 < 3:1（如 70/30）
- 中度：比例 3:1 到 20:1（如 95/5）
- 重度：比例 > 20:1（如 99/1）

## 第 2 步：选择正确的评估指标

对于不平衡数据，优先使用精确率 / 召回率 / F1 而不是准确率。根据你的问题选择：

| 场景 | 主要指标 | 次要指标 |
|------|---------|---------|
| 漏检正例代价极高（欺诈、疾病） | 召回率 | F2 分数 |
| 误报代价极高（垃圾邮件过滤、推荐） | 精确率 | F0.5 分数 |
| 两者大致同等重要 | F1 分数 | MCC |
| 需要单一排序指标 | AUPRC | AUC-ROC |
| 需要跨数据集比较 | MCC | AUPRC |

## 第 3 步：选择再平衡策略

### 按不平衡程度

| 不平衡程度 | 首选方案 | 备选方案 | 避免 |
|-----------|---------|---------|------|
| 轻度（< 3:1） | 类别权重 | 阈值优化 | 过采样（不必要） |
| 中度（3:1 ~ 20:1） | SMOTE + 类别权重 | 在此基础上做阈值优化 | 欠采样（数据损失太多） |
| 重度（> 20:1） | SMOTE + 类别权重 + 阈值 | 平衡装袋集成 | 单独使用欠采样 |

### 按数据集规模

| 数据集规模 | 推荐策略 | 原因 |
|-----------|---------|------|
| < 1000 样本 | 过采样或 SMOTE | 不能承受丢弃多数类数据 |
| 1000 ~ 10000 样本 | SMOTE + 阈值优化 | 有足够的少数类样本做 k 近邻 |
| > 10000 样本 | 类别权重或欠采样 | 训练快，少数类样本充足 |

## 第 4 步：应用技术

### 类别权重（始终先试这个）
- 使用 scikit-learn：`class_weight='balanced'`
- 不需要修改数据
- 适用于任何基于损失的模型
- 期望上等价于过采样

### SMOTE
- 仅应用于训练数据（永远不要用于测试/验证集）
- 使用 k=5 近邻（默认值）
- 与类别权重组合使用效果更好
- 注意边界附近的噪声合成点

### 阈值优化
- 训练模型，在验证集上获取预测概率
- 从 0.05 到 0.95 遍历阈值
- 选择使目标指标最大的阈值
- 始终在验证集上优化，不要在测试集上

## 第 5 步：正确验证

- 使用分层交叉验证（保持每折中的类别比例）
- 在原始（未重采样）测试集上报告指标
- 永远不要在划分之前应用 SMOTE——仅在训练折上操作
- 与"始终预测多数类"的基线进行比较

## 第 6 步：避免常见错误

- 在训练/测试划分之前应用 SMOTE（数据泄露）
- 使用准确率作为评估指标
- 不先尝试类别权重（最简单的方法，通常就够用）
- 过采样后再做交叉验证（合成点会在各折之间泄露）
- 忽略阈值优化（免费性能提升，无需重新训练）
- 在小数据集上使用随机欠采样（丢弃太多有用数据）

## 快速决策树

1. 不平衡比例 < 3:1？ → 仅使用类别权重
2. 数据集 > 10000 样本？ → 类别权重 + 阈值优化
3. 数据集 < 1000 样本？ → SMOTE + 类别权重
4. 其他情况 → SMOTE + 类别权重 + 阈值优化
5. 仍然不够好？ → 平衡装袋集成
