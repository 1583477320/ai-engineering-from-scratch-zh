# 潜在扩散与 Stable Diffusion

> DDPM 在像素空间做扩散太慢。潜在扩散在 VAE 的潜空间做扩散——用 64×64 的"压缩图"代替 512×512 的原始图——速度提升 10-50 倍，质量几乎不变。

## 1. 问题

DDPM 在 512×512 的像素空间上做扩散——需要 1000 步去噪，每步都在 512×512 的维度上计算——极其缓慢。解决方案：**先用 VAE 将图像压缩到 64×64 的潜空间，在那里做扩散。**

Stable Diffusion（Rombach et al., 2022）将潜在扩散商品化——在消费级 GPU 上生成高质量图像。OpenAI、Google、Meta 的图像生成都建立在这个架构之上。

---

## 2. 概念

### 2.1 潜在扩散的两阶段

```
训练阶段:
  原始图像 → [VAE 编码器] → 潜向量 z → [扩散加噪] → 预测噪声 ε_θ(z_t, t)

生成阶段:
  随机噪声 z_T → [U-Net 去噪 T 步] → 潜向量 z_0 → [VAE 解码器] → 图像
```

### 2.2 Stable Diffusion 的三大组件

| 组件 | 作用 | 维度 |
|---|---|---|
| **VAE 编码器** | 将图像压缩到潜空间 | 512×512→64×64×4 |
| **U-Net + 交叉注意力** | 在潜空间去噪 + 条件化 | 64×64×4 |
| **VAE 解码器** | 将潜向量解码回图像 | 64×64×4→512×512 |

### 2.3 条件化——文本如何控制生成

```
文本 → CLIP 编码器 → 文本嵌入 → 交叉注意力 → U-Net
                                        ↓
                              每个时间步关注不同位置
```

交叉注意力让 U-Net 在去噪过程中"看"文本描述——决定在哪个区域生成什么内容。

---

## 🔑 关键术语

| 术语 | 含义 |
|---|---|
| 潜在扩散 | 在 VAE 的潜空间而非像素空间做扩散——快 10-50 倍 |
| U-Net | 扩散模型的去噪网络——编码-解码 + 跳跃连接 |
| 交叉注意力 | 文本条件输入的方式——让去噪过程"参考"文本描述 |
| CLIP | OpenAI 的对比学习模型——将图像和文本映射到同一嵌入空间 |

---

## 📚 小结

潜在扩散将扩散模型的速度提升 10-50 倍——通过在 VAE 的 64×64 潜空间而非 512×512 像素空间做扩散。Stable Diffusion 将这个架构商品化——VAE + U-Net + CLIP 交叉注意力。文本通过 CLIP 编码器注入条件信息。
