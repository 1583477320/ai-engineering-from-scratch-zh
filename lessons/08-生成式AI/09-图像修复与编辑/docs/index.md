# 图像修复与编辑

> 修复（inpainting）填补缺失区域；外扩（outpainting）扩展边界；编辑修改特定区域。三者都是"条件扩散+掩码"的变体。

**类型：** 概念课 | **语言：** Python | **前置知识：** 阶段 08 · 07（潜在扩散）| **时间：** ~45 分钟

## 🎯 学习目标

- [ ] 理解 inpainting 的掩码机制——在二进制掩码下，只对未遮盖区域进行去噪
- [ ] 解释 outpainting 如何通过上下文扩展实现——从已知像素向外生成

## 1. 问题

Inpainting：给定图像和掩码，修复缺失区域（去除水印、修复损坏照片、填充空白）。
Outpainting：从已知图像向外扩展（扩展画布、修复边缘）。
图像编辑：用文本指令修改图像特定区域（改变颜色、更换背景、调整风格）。

**核心原理相同：** 在 U-Net 的注意力中用掩码信号告诉模型"哪些区域需要生成、哪些区域保持原样"。

---

## 2. 概念

### 2.1 Inpainting 掩码机制

```
原图像 + 二进制掩码 → VAE 编码 → U-Net 去噪（掩码=0的位置保持原样）→ VAE 解码 → 完整图像
```

掩码告诉模型："只在掩码=1的区域生成新内容，掩码=0的区域保持原样。"

### 2.2 Outpainting

从已知图像的边缘向外生成——相当于在图像的上下文中进行"条件生成"。通常使用 U-Net 的交叉注意力——让模型"看到"边缘像素，生成连续的内容。

### 2.3 InstructPix2Pix（2023）

用文本指令编辑图像。"把天空变蓝"、"让猫戴帽子"。核心：CLIP 文本编码 → 条件注入 U-Net → 只对相关区域进行修改。

---

## 🔑 关键术语

| 术语 | 含义 |
|---|---|
| Inpainting | 修复图像中的缺失区域——掩码指导生成 |
| Outpainting | 从已知图像向外扩展——需要上下文感知 |
| InstructPix2Pix | 用文本指令编辑图像——"把 X 改成 Y" |

---

## 📚 小结

Inpainting/Outpainting/Edit 都是"条件扩散+掩码"的变体——掩码告诉模型哪些区域需要生成。InstructPix2Pix 用 CLIP 文本编码+掩码实现了文本指导的图像编辑。2026 年这些功能被集成进 DALL-E 3、Midjourney、Stable Diffusion 的 UI 中。
