# 条件 GAN 与 Pix2Pix 实践指南

> 本指南帮助你在实际项目中使用条件 GAN 技术。从选型决策到训练调参，覆盖工业界常见的场景和陷阱。

---

## 1. 什么时候用条件 GAN？

| 场景 | 推荐方案 | 理由 |
|---|---|---|
| 配对图像翻译（边缘→照片） | Pix2Pix | 有配对数据，Pix2Pix 是最直接的选择 |
| 无配对风格迁移 | CycleGAN | 没有配对数据，循环一致性损失解决问题 |
| 图像修复/补全 | 条件 GAN + L1 | 条件输入是带孔洞的图像，输出是修复后的图像 |
| 语义分割着色 | Pix2Pix | 语义分割图→彩色照片，天然配对 |
| 文本→图像 | 条件 GAN（CLIP 引导） | 文本嵌入作为条件输入 |
| 超分辨率 | SRGAN | 条件输入是低分辨率图像 |

### 不适合用条件 GAN 的场景

| 场景 | 推荐替代 | 理由 |
|---|---|---|
| 高质量文生图 | 扩散模型（Stable Diffusion） | 条件 GAN 的生成质量和多样性不如扩散模型 |
| 文本生成 | 自回归模型 | 文本的离散性不适合 GAN |
| 无条件生成 | StyleGAN3 或扩散模型 | 不需要条件时，用专门的无条件架构更好 |

---

## 2. Pix2Pix vs CycleGAN 选型

```
你有配对数据吗？
├── 是 → 用 Pix2Pix（更简单，训练更稳定）
│         └── 数据量少（< 1000 对）→ 考虑数据增强
└── 否 → 用 CycleGAN（需要两个 GAN + 循环一致性损失）
          └── 两个域差异太大 → 考虑加入语义分割等辅助信息
```

## 3. 超参数调优指南

### 3.1 学习率

| 参数 | 推荐值 | 调优范围 | 说明 |
|---|---|---|---|
| G 学习率 | 2e-4 | 1e-4 ~ 5e-4 | 生成器需要较快的学习速度 |
| D 学习率 | 1e-4 | 5e-5 ~ 2e-4 | 判别器通常比生成器慢 2-5 倍 |
| β1 | 0.5 | 0.4 ~ 0.6 | Adam 的动量参数，0.5 是 GAN 的经典值 |
| β2 | 0.999 | 0.99 ~ 0.999 | 二阶矩估计 |

### 3.2 损失权重

| 参数 | 推荐值 | 作用 |
|---|---|---|
| LAMBDA_L1 | 100 | L1 损失权重，越大生成图像越接近目标（更清晰但可能更模糊） |
| LAMBDA_L2 | 可选 | L2 损失，比 L1 更平滑但可能更模糊 |

### 3.3 训练策略

```python
# 1. 标签平滑（Pix2Pix 论文中不推荐，但有时有用）
real_label = torch.ones(...) * 0.9  # 不是 1.0

# 2. 噪声注入（防止判别器过强）
fake = generator(condition) + torch.randn_like(fake) * 0.05

# 3. 学习率衰减（最后 100 轮线性衰减到 0）
scheduler_g = torch.optim.lr_scheduler.LinearLR(opt_g, start_factor=1.0, end_factor=0.0, total_iters=100)
```

## 4. PatchGAN 的 patch 大小选择

| Patch 大小 | 覆盖范围 | 效果 |
|---|---|---|
| 70x70（默认） | 中等 | 平衡全局结构和局部纹理 |
| 28x28 | 小 | 更关注局部纹理，可能忽略全局结构 |
| 全图（1x1） | 整图 | 等价于普通判别器，丢失 PatchGAN 优势 |

实际使用中，70x70 patch 是经验最优值。修改方法：调整判别器的层数。

## 5. 常见问题排查

| 现象 | 可能原因 | 解决方案 |
|---|---|---|
| 生成图像全黑 | L1 权重太大，G 只学到均值 | 降低 LAMBDA_L1（如 10 → 100） |
| 生成图像模糊 | L1 损失主导，对抗损失不足 | 增加 LAMBDA_L1 或降低 D 学习率 |
| 生成图像有棋盘格伪影 | 转置卷积步长与核大小不匹配 | 检查 UNetUp 的 ConvTranspose2d 参数 |
| D 损失降到 0 | D 太强，G 梯度消失 | 降低 D 学习率，加 Dropout，用标签平滑 |
| 训练不稳定（loss 震荡） | G 和 D 容量不平衡 | 检查两者参数量是否接近 |
| CycleGAN 训练后颜色偏移 | 循环一致性损失不够 | 增加 LAMBDA_CYCLE（如 5 → 10） |

## 6. 从 Pix2Pix 到 ControlNet 的演进

2026 年，Pix2Pix 的思想被 ControlNet（2023）继承和扩展：

| 特性 | Pix2Pix | ControlNet |
|---|---|---|
| 条件输入 | 一张条件图像 | 多种条件（边缘、深度、姿态、语义分割） |
| 生成器 | U-Net GAN | 基于预训练扩散模型 |
| 训练数据 | 需要配对数据 | 可以用更少的配对数据 |
| 质量 | 受限于 GAN 的模式坍塌 | 扩散模型的质量更高 |
| 推理速度 | 快（单次前向传播） | 慢（需要多步去噪） |

**关键洞察：** ControlNet 的核心思想——"在预训练模型上添加条件控制"——可以追溯到 Pix2Pix 的条件 GAN。理解 Pix2Pix 是理解 ControlNet 的基础。
