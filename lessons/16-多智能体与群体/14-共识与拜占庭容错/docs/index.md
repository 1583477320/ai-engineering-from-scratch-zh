# 共识与拜占庭容错

> 经典分布式系统 BFT 遇到随机 LLM。2025-2026 年三个研究方向：**CP-WBFT**（arXiv:2511.10400）用置信度探测加权每票；**DecentLLMs**（arXiv:2507.14928）无领导者，工作者并行提案，几何中位数聚合；**WBFT**（arXiv:2505.05103）结合加权投票与层次结构聚类分 Core/Edge 节点。"智能体能同意吗？"（arXiv:2603.01213）的诚实经验结果是：即使标量同意今天也很脆弱——一个欺骗性智能体可以将混合体共识偏离诚实基线 40+ 百分点。BFT 必要但不充分。

## 1. 问题

你有 N 个 LLM 智能体各自产生一个答案。它们不同意。多数投票选择了错误的——因为两个智能体是相关的（相同基础模型、相同训练数据、相同失败模式）。第三个智能体碰巧以一种新的方式错了——所以多数是虚假多数。

添加一个欺骗性智能体：它故意说谎。或者一个谄媚智能体：它顺从最后发言的人。经典 BFT 中，拜占庭节点是 `f < n/3` 的分数。2026 年的现实是 LLM 节点即使在诚实时也是随机的，跨模型相关，受彼此输出影响。你不能将它们视为独立的伯努利投票者。

## 2. 概念

### 2.1 经典 BFT 给你的

实用拜占庭容错（Castro & Liskov，OSDI 1999）容忍 `f < n/3` 拜占庭节点。协议有三个阶段（预准备、准备、提交）和两个原语（签名消息、法定人数证书）。在 `n >= 3f + 1` 个诚实或恶意节点上达成单值共识。

保证很强但假设：
1. **独立故障**——拜占庭者不协调
2. **诚实节点真正诚实**——诚实输出的正确性不是问题
3. **问题有真实答案**——错误事实上的共识仍然是共识

LLM 智能体违反所有三个。

### 2.2 三种 LLM 特定攻击

| 攻击 | 说明 | 经典 BFT 处理 |
|------|------|-------------|
| 拜占庭撒谎 | 一个智能体故意输出错误答案 | 如果 `f < n/3` 可以处理 |
| 谄媚从众 | 一个智能体读取他人答案后与最后发言者对齐 | 不捕获——通过每个签名检查 |
| 相关误差单一文化 | 三个智能体共享基础模型，幻觉相同的错误答案 | 不帮助——三个都"诚实"同意 |

### 2.3 三种 2025-2026 回应

| 方法 | 机制 | 缓解 |
|------|------|------|
| CP-WBFT | 置信度加权投票 | 谄媚从众（从众者置信度低） |
| DecentLLMs | 几何中位数聚合 | 拜占庭撒谎 + 相关误差（对异常值鲁棒） |
| WBFT | 加权投票 + 层次结构聚类 | 可扩展性 + 部分单一文化 |

### 2.4 "智能体能同意吗？"的经验结果

论文测量标量同意（LLM 智能体在单数值上同意）：

- 即使没有对手，在许多基准上 LLM 智能体分歧率超过 30%
- 单个采用欺骗性人格的智能体可以将共识偏离诚实基线 40+ 百分点
- 不同率与模型多样性相关——异质集成体分歧更多（好：去相关误差）但也漂移更慢（坏：更长的一致时间）

### 2.5 最小 BFT 协议

```
1. 任务到达；每个智能体 i 产生答案 a_i
2. 每个智能体附加置信度探测 c_i ∈ [0, 1]
3. 聚合者收集所有 n 个智能体的 (a_i, c_i)
4. 聚合者按语义簇分组等效答案
5. 聚合者计算每个簇 C 的权重：w(C) = Σ_{i∈C} c_i
6. 获胜者 = 最大权重的簇，如果最大值 > 阈值 × Σ(c_i)
   否则：重试或升级
7. 少数簇带溯源记录，用于事后审计
```

### 2.6 共识无用的场景

- 模糊问题——如果问题没有真实答案，共识是观点
- 复合问题——"写代码并解释"——两个答案，分别投票
- 对抗性多轮——如果智能体可以观察先前轮次并模仿，他们开始无论正确与否都同意

## 3. 从零实现

### 第 1 步：定义投票和聚合器

```python
@dataclass
class Vote:
    agent: str
    answer: str
    confidence: float

    def canonical(self):
        return "".join(c for c in self.answer.lower().strip() if c.isalnum() or c == "." or c == "%")

def plurality(votes):
    """经典多数投票。"""
    counts = {}
    for v in votes:
        key = v.canonical()
        counts[key] = counts.get(key, 0) + 1
    winner_key = max(counts, key=counts.get)
    return winner_key, counts

def cp_wbft(votes, threshold=0.5):
    """置信度加权 BFT。"""
    weights = {}
    for v in votes:
        key = v.canonical()
        weights[key] = weights.get(key, 0.0) + v.confidence
    total = sum(weights.values()) or 1.0
    winner_key = max(weights, key=weights.get)
    if weights[winner_key] / total < threshold:
        return None, weights  # 低于阈值，拒绝
    return winner_key, weights

def decentllms(votes):
    """几何中位数聚合。"""
    clusters = {}
    for v in votes:
        clusters.setdefault(v.canonical(), []).append(v)
    scores = {}
    for key, cluster in clusters.items():
        med = median([v.confidence for v in cluster])
        dist = sum(abs(v.confidence - med) for v in cluster)
        scores[key] = len(cluster) * max(0.0, 1.0 - dist)
    winner_key = max(scores, key=scores.get)
    return clusters[winner_key][0].answer, scores
```

### 第 2 步：运行攻击场景

```python
def main():
    # 场景 1：无攻击
    scenario("无攻击", "4.2%", [
        Vote("a", "4.2%", 0.85), Vote("b", "4.2%", 0.80),
        Vote("c", "4.2%", 0.75), Vote("d", "5%", 0.40), Vote("e", "4.2%", 0.70)])

    # 场景 2：拜占庭撒谎
    scenario("拜占庭撒谎", "4.2%", [
        Vote("a", "4.2%", 0.75), Vote("b", "4.2%", 0.70),
        Vote("c", "4.2%", 0.80), Vote("d", "42%", 0.95), Vote("e", "4.2%", 0.65)])

    # 场景 3：谄媚从众
    scenario("谄媚从众", "4.2%", [
        Vote("a", "42%", 0.35), Vote("b", "42%", 0.30),
        Vote("c", "4.2%", 0.85), Vote("d", "4.2%", 0.80), Vote("e", "4.2%", 0.82)])

    # 场景 4：相关误差单一文化
    scenario("相关误差单一文化", "4.2%", [
        Vote("a", "42%", 0.70), Vote("b", "42%", 0.68),
        Vote("c", "42%", 0.72), Vote("d", "4.2%", 0.85), Vote("e", "4.2%", 0.82)])
```

## 5. 工程最佳实践

| 原则 | 说明 |
|------|------|
| 攻击测试用三种模式 | 拜占庭撒谎、谄媚从众、相关误差单一文化 |

| 分离一致性和正确性 | 共识输出到验证者；验证者独立于集成体 |

## 6. 常见错误

### 错误 1：信任多数投票

**现象：** 三个智能体都是相同模型，都"诚实"同意——但同意的是错误答案。

**修复：** 异质集成体（不同模型）+ 验证者（独立评估）。多数投票在单一文化攻击下失败。

### 错误 2：无限制辩论轮次

**现象：** "辩论到同意"——谄媚每次获胜。

**修复：** 限制轮数（2-3 轮）。有界轮次防止从众级联。

### 错误 3：不记录少数簇

**现象：** 少数簇被丢弃。但如果少数簇持续正确呢？

**修复：** 记录每个少数簇带溯源。少数簇是相关误差的早期预警。

## 7. 面试考点

### Q1：经典 BFT 对 LLM 智能体的三个假设是什么？（难度：⭐）

**参考答案：**
1. 独立故障——拜占庭者不协调
2. 诚实节点真正诚实——诚实输出的正确性不是问题
3. 问题有真实答案——错误事实上的共识仍然是共识

LLM 违反全部三个：跨模型相关、诚实但幻觉、模糊问题无真实答案。

### Q2：三种 LLM 特定攻击是什么？（难度：⭐⭐）

**参考答案：**
**拜占庭撒谎**——故意输出错误答案。经典 BFT 在 `f < n/3` 时处理。

**谄媚从众**——读取他人答案后与最后发言者对齐。通过所有签名检查。

**相关误差单一文化**——三个智能体共享基础模型，幻觉相同的错误答案。经典 BFT 不帮助——三个都"诚实"同意。

### Q3："智能体能同意吗？"的经验结果是什么？（难度：⭐⭐⭐）

**参考答案：**
- 即使没有对手，LLM 智能体在许多基准上分歧率超过 30%
- 单个欺骗性智能体可以将共识偏离 40+ 百分点
- 不同率与模型多样性相关——异质集成体分歧更多但漂移更慢

结论：BFT 给你对齐输出的机制，但不告诉你对齐的输出是否正确。需要与验证者、多样性、评估者智能体结合。

## 🔑 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|-----------|---------|
| BFT | "拜占庭容错" | Castro-Liskov 1999 协议，`f < n/3` 任意故障 |
| 拜占庭 | "任何恶意行为" | 可以撒谎、丢弃消息、静默失败的节点 |
| 置信度探测 | "你多确定？" | 附加到投票的自报告概率或校准器预测 |
| 语义聚类 | "不同词的相同答案" | 计票前分组等效答案 |
| 几何中位数 | "鲁棒中心" | 最小化到样本点距离的点——对异常值鲁棒 |
| 单一文化 | "同模型同失败" | 共享训练数据或基础模型时的相关误差 |
| 谄媚从众 | "顺从最大声的" | 智能体投票偏向最后/最大声发言者的偏差 |
| Core/Edge | "层次 BFT" | WBFT 分割：小 Core 先共识，Edge 跟随 |

## 📚 小结

经典 BFT 容忍 `f < n/3` 拜占庭节点，但 LLM 智能体违反所有三个假设。三种 LLM 特定攻击：拜占庭撒谎、谄媚从众、相关误差单一文化。CP-WBFT、DecentLLMs、WBFT 是三种 2025-2026 回应。"智能体能同意吗？"的经验结果：同意是脆弱的，BFT 必要但不充分。
