# MARL——MADDPG、QMIX、MAPPO

> 多智能体强化学习的经典算法，仍影响着 2026 年的 LLM 智能体系统。MADDPG 引入了集中训练分散执行。QMIX 是带单调混合网络的价值分解。MAPPO 是带集中值函数的 PPO——2026 年的默认合作 MARL 基线。本课从一个小型网格世界玩具构建它们，在接触 LLM 智能体训练之前将三个思想落地为肌肉记忆。

## 1. 问题

多智能体系统越来越多地训练智能体间协调的策略：何时延后、何时行动、调用哪个对等体。告诉你如何训练这些策略的文献是多智能体强化学习（MARL）。但独立 RL 是非平稳的；集中式 RL 不缩放且违反执行约束。CTDE 取得两者最佳。

---

## 2. 概念

### 2.1 三个算法

**MADDPG（2017）：** 每个智能体有 actor（自己观察到行动）和 critic（看到所有观察和所有行动）。训练时 critic 集中，部署时只跑 actor。失败模式：critic 输入随 N 增长，不缩放。

**QMIX（2018）：** 合作场景。整体值函数是各智能体 Q 值的单调函数。单调性保证 argmax 可分解——每个智能体独立选择 argmax。在星际争霸合作微操上获胜。

**MAPPO（2022）：** PPO 加集中值函数。多智能体 PPO 在粒度世界、星际争霸、谷歌研究足球上匹配或击败离策略 MARL 方法。最少超参数调优。2026 年默认基线。

### 2.2 CTDE 作为设计模式

即使不训练，CTDE 也是有用的架构模式：设计时假设全团队可见，运行时强制分散执行。迫使你思考部分可观察性。

## 5. 常见错误

### 错误 1：独立 RL

智能体互相把对方视为环境的一部分——非平稳。一个改变，另一个的基础改变。

### 错误 2：忽略 MAPPO

MAPPO 匹配或击败离策略方法。2026 年默认基线。

### 错误 3：在 LLM 系统中跳过 CTDE

CTDE 即使不训练也是有用的设计模式——设计时全团队可见，运行时只看到自己。

---

## 6. 面试考点

### Q1：CTDE 是什么？（难度：⭐）

集中训练分散执行。训练时全局信息，部署时局部策略。

### Q2：三者的区别是什么？（难度：⭐⭐）

MADDPG：每个智能体有集中 critic。QMIX：单调混合各智能体 Q 值。MAPPO：PPO 加集中值函数。

## 小结

MADDPG（CTDE 模式）、QMIX（单调价值分解）、MAPPO（集中值函数）是 MARL 的三个核心算法。MAPPO 是 2026 年默认合作基线。CTDE 即使不训练也是有用的架构设计模式。
