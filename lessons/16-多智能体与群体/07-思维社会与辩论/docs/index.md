# 思维社会与多智能体辩论

> Minsky 1986 年的前提——智能是专家的社会——每十年被重新发现一次。2023 年 Du 等人将其变为具体算法：多个 LLM 实例提出答案、读取彼此的答案、批评并更新。经过 N 轮后它们收敛到一个在六项推理和事实性任务上击败零样本 CoT 和反思的共识。两个发现重要：**多个智能体**和**多轮交换**各自独立贡献。社会击败单智能体独白；多轮交换击败一次性投票。

## 1. 问题

自一致性——从一个模型采样多次然后取多数答案——是最便宜的推理改进。它有效但饱和很快。你可以加倍样本而看不到另一个有意义的跳跃。

辩论打破了饱和。不是 N 个独立样本，N 个智能体读取彼此的推理并修改。样本间的相关性下降（它们不再是 i.i.d.），收敛点通常在 i.i.d. 投票自信地错误的地方是正确的。

## 2. 概念

### 2.1 Du 等人 2023 年算法

来自 arXiv:2305.14325（ICML 2024）：

1. N 个智能体各自产生问题的初始答案
2. 第 r=2..R 轮：每个智能体看到其他智能体第 r-1 轮的答案，被要求"考虑这些，给出你的更新答案"
3. R 轮后，对最终答案进行多数投票

在 MMLU、GSM8K、传记、MATH 和事实性基准上测试。辩论一致击败 CoT 和自我反思。

### 2.2 两个独立旋钮

同一篇论文的消融实验：

- **仅智能体数量**（1 轮，N 个的多数投票）在大多数任务上击败单智能体，但饱和
- **仅轮数**（1 个智能体看到自己先前的推理）几乎没帮助——反思的已知弱点
- **两者一起**产生大跳跃。多智能体之间的多轮交换驱动增益

### 2.3 为什么有效

两个机制：

1. **暴露分歧**——当一个智能体看到另一个不同结论的推理链时，它必须证明或更新。无论如何，第 r+1 轮的上下文比第 r 轮更丰富
2. **相关误差减少**——自一致性中所有样本来自同一模型，误差相关——你平均到一个自信错误的答案。辩论进一步去相关

### 2.4 异质辩论

A-HMAD 和相关后续使用**不同基础模型**给不同智能体。Llama + Claude + GPT 辩论减少了单一文化崩溃，因为一个模型家族的相关误差不被其他家族共享。

缺点：弱模型参与辩论可能将共识拖向其错误答案。

### 2.5 NLSOM——129 智能体扩展

诸葛等人将此想法扩展到 129 成员社会。结果：专业化和自组织随规模涌现，系统在视觉问答等任务上超越单智能体。

### 2.6 失败模式

| 失败 | 说明 | 缓解 |
|------|------|------|
| 谄媚级联 | 所有智能体都顺从听起来最自信的 | 提示对抗性角色（"一个智能体必须争辩反方"） |
| 主题漂移 | 多轮辩论偏离原始问题 | 每轮重新注入问题 |
| 计算爆炸 | N 智能体 × R 轮 = N·R 次 LLM 调用，上下文增长 | 限制轮数和智能体数 |

## 3. 从零实现

### 第 1 步：定义辩论智能体

```python
@dataclass
class DebateAgent:
    name: str
    answer: float
    confidence: float
    history: list[float] = field(default_factory=list)

    def revise(self, others):
        """基于自己和他人答案的加权平均更新。"""
        weights = [self.confidence] + [o.confidence for o in others]
        values = [self.answer] + [o.answer for o in others]
        total_w = sum(weights)
        self.answer = sum(w * v for w, v in zip(weights, values)) / total_w
        self.confidence = min(self.confidence * 1.05, 1.0)
        self.history.append(self.answer)
```

### 第 2 步：实现辩论循环

```python
TRUE_ANSWER = 42.0

def run_debate(agents, rounds):
    for a in agents:
        a.history.append(a.answer)

    for r in range(1, rounds + 1):
        updates = [(a, [o for o in agents if o is not a]) for a in agents]
        for a, others in updates:
            a.revise(others)
        # 记录每轮结果
        mean = sum(a.answer for a in agents) / len(agents)
        err = abs(mean - TRUE_ANSWER)
        print(f"  轮 {r}: 均值={mean:.2f}, 误差={err:.2f}")
```

### 第 3 步：运行对比

```python
def main():
    # 对比基线：单轮多数投票
    baseline = [DebateAgent("A", 38.0, 0.6), DebateAgent("B", 42.5, 0.8), DebateAgent("C", 51.0, 0.4)]
    control_mean = sum(a.answer for a in baseline) / len(baseline)
    print(f"控制（第 0 轮均值）: {control_mean:.2f}")

    # 3 智能体 × 3 轮
    team3 = [DebateAgent("A", 38.0, 0.6), DebateAgent("B", 42.5, 0.8), DebateAgent("C", 51.0, 0.4)]
    run_debate(team3, rounds=3)

    # 3 智能体 × 5 轮（边际收益递减）
    team5 = [DebateAgent("A", 38.0, 0.6), DebateAgent("B", 42.5, 0.8), DebateAgent("C", 51.0, 0.4)]
    run_debate(team5, rounds=5)
```

## 4. 工业工具

### 4.1 辩论配置建议

| 参数 | 推荐 | 理由 |
|------|------|------|
| 轮数上限 | 3 | Du 等人显示 3 轮捕获大部分增益 |
| 智能体数量上限 | 5 | 超过 5 后上下文膨胀和成本主导 |
| 异质性 | 默认 | 至少两个不同基础模型 |
| 对抗性槽位 | 一个智能体被提示始终争辩 | 打破谄媚 |
| 每轮日志 | 必须 | 辩论系统隐藏中间轮无法调试或审计 |

## 6. 常见错误

### 错误 1：谄媚级联

**现象：** 所有智能体顺从听起来最自信的，不论正确性。

**修复：** 提示一个智能体扮演对抗性角色——"你必须争辩反方"。

### 错误 2：主题漂移

**现象：** 多轮辩论后偏离原始问题。

**修复：** 每轮重新注入原始问题。

### 错误 3：计算爆炸

**现象：** N 智能体 × R 轮 = N·R 次 LLM 调用，上下文增长。5 智能体 × 5 轮 = 25 次调用。

**修复：** 限制轮数（≤3）和智能体数（≤5）。

## 7. 面试考点

### Q1：辩论为什么比自一致性更好？（难度：⭐）

**参考答案：**
自一致性中所有样本来自同一模型，误差相关——你平均到一个自信错误的答案。辩论中智能体读取彼此的推理，误差去相关。多轮交换进一步去相关。

两个独立贡献：智能体数量（N 个智能体的多数投票击败单智能体）+ 轮数（多轮交换进一步提升）。两者缺一不可。

### Q2：Du 等人的消融实验发现了什么？（难度：⭐⭐）

**参考答案：**
- 仅智能体数量（1 轮）：在大多数任务上击败单智能体，但饱和
- 仅轮数（1 个智能体看自己先前推理）：几乎没帮助——反思的已知弱点
- 两者一起：产生大跳跃

多智能体之间的多轮交换驱动增益。

### Q3：谄媚级联是什么？如何缓解？（难度：⭐⭐）

**参考答案：**
所有智能体顺从听起来最自信的智能体，不论正确性。辩论退化为最大声者独白。

缓解：提示一个智能体扮演对抗性角色——"你必须争辩反方，不论你同意还是不同意"。这打破了谄媚循环。

## 📚 小结

Du 等人的辩论算法：N 个智能体提出答案→读取彼此→修改→多数投票。两个独立旋钮：智能体数量和轮数——两者都必要。多轮交换击败一次性投票。三个失败模式：谄媚级联、主题漂移、计算爆炸。轮数上限 3，智能体数上限 5，异质性默认。
