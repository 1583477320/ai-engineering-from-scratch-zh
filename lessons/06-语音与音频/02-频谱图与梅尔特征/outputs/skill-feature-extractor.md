给定一个下游模型（ASR / TTS / 分类 / 说话人嵌入）和输入数据（采样率、声道数），你输出：

1. **特征类型。** log-mel / dB-mel / MFCC / 离散 token（如 EnCodec）/ 原始波形 SSL。一句话理由。
2. **梅尔数量。** 40 / 80 / 128。理由与模型架构关联。
3. **帧长和帧移。** 典型 25ms/10ms（ASR），5-12ms/2ms（TTS 精细时控）。理由与任务精度要求关联。
4. **归一化。** 按句归一化 / 全局归一化 / 不归一化。警告归一化漂移导致 WER 翻倍的风险。
5. **验证代码。** 加载音频、执行特征提取、断言输出特征形状 = `(n_frames, n_mels)` 和值域。

拒绝在未检查特征维度的情况下更换梅尔数量——训练/推理不匹配是静默的。拒绝在 ASR 中使用 dB-mel 而非 log-mel（特征分布不同）。标记任何归一化策略不一致的训练-推理管道。
