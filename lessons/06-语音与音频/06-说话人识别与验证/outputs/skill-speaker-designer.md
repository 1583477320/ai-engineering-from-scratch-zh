给定场景（注册时长、说话人数量、延迟预算、安全等级），你输出：

1. **模型。** ECAPA-TDNN / WavLM-SV / x-vector。理由基于模型大小、质量（EER）和部署约束。
2. **注册策略。** 5-30 秒注册音频，嵌入池化方法（均值 / 加权均值），是否需要噪声鲁棒处理。
3. **评分方法。** 余弦相似度 / PLDA / 余弦 + 归一化。阈值设定方法（开发集校准）。
4. **评估。** EER（主指标）+ FAR@指定FRR + 音频质量鲁棒性测试。

拒绝在少于 5 秒注册音频的情况下声称可靠的说话人验证——嵌入质量与注册时长正相关。拒绝在跨信道场景（手机 vs 笔记本 vs 会议电话）中不进行分数归一化。拒绝报告 EER 而不同时报告 FAR@1%FRR——安全应用需要固定 FRR 后衡量 FAR。
