# 投标评估提示词模板

> 用于多智能体系统中，让 LLM 智能体评估自身是否适合执行某个任务，并以结构化格式输出投标信息。

---

## 使用说明

将以下提示词发送给投标者智能体，替换 `{变量}` 部分为实际值。

---

## 提示词

```
你是一个多智能体系统中的投标者。请评估你是否适合执行以下任务。

## 任务信息
- 任务描述：{task_description}
- 所需技能：{required_skill}
- 优先级：{priority}
- 预算上限：{max_budget} 算力单位

## 你的能力
- 技能列表：{agent_skills}
- 当前负载：{current_load}（0=空闲，1=满载）
- 历史成功率：{history_success}

## 评估要求
请从以下维度评估：
1. 技能匹配度：你在该技能上的能力评分（0~1）
2. 预估成本：完成该任务需要的算力单位
3. 预估耗时：完成该任务需要的时间（秒）
4. 投标意愿：你是否愿意投标（true/false）
5. 理由：一句话说明你的评估依据

## 输出格式
请严格返回以下 JSON 格式，不要添加任何其他内容：

{
  "willing": true/false,
  "skill_match": 0.0~1.0,
  "estimated_cost": 数值,
  "estimated_time": 数值,
  "reason": "一句话理由"
}
```

---

## 示例

**输入：**
- 任务描述：分析上季度销售数据
- 所需技能：数据分析
- agent_skills：{"数据分析": 0.9, "文本写作": 0.3}
- current_load：0.2

**输出：**
```json
{
  "willing": true,
  "skill_match": 0.9,
  "estimated_cost": 39.2,
  "estimated_time": 1.4,
  "reason": "数据分析是我的核心技能，当前负载较低，可以高效完成"
}
```

---

## 注意事项

- 要求智能体返回**结构化 JSON**，而不是自然语言描述
- 如果智能体的技能匹配度 < 0.3，应该返回 `"willing": false`
- 预估成本应考虑当前负载——负载越高，成本越高
- 理由字段用于调试和审计，帮助理解智能体的决策依据
