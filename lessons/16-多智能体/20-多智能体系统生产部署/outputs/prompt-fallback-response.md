# 降级响应提示词模板

> 用于多智能体系统中，当某个智能体失败时，生成有意义的降级响应而非错误信息。

---

## 使用说明

当智能体调用失败（超时、API 错误、返回异常）时，将以下提示词发送给备用智能体或主模型，生成降级响应。

---

## 提示词

```
## 系统状态
智能体 "{agent_name}" 在执行任务时遇到了问题：
- 错误类型：{error_type}
- 错误详情：{error_detail}
- 已重试次数：{retry_count}

## 任务信息
- 原始任务：{task_description}
- 已完成的步骤：{completed_steps}
- 剩余的步骤：{remaining_steps}

## 降级策略
请根据以下策略生成降级响应：

1. 如果已有部分结果（如数据分析已完成，但报告撰写失败），基于已有结果生成简化版输出
2. 如果完全没有结果，生成一个说明性的模板响应，告诉用户系统遇到了什么问题
3. 响应必须包含：(a) 已完成的工作（如果有），(b) 未能完成的工作，(c) 建议的下一步

## 输出要求
- 保持专业和友好的语气
- 不要暴露技术细节（如 API 错误码）
- 提供可操作的建议
- 响应长度不超过 200 字
```

---

## 示例

**输入：**
- agent_name：报告撰写者
- error_type：超时
- task_description：生成市场分析报告
- completed_steps：数据分析（已完成）
- remaining_steps：报告撰写、图表生成

**输出：**
```
市场分析的数据采集和分析已完成。报告撰写环节遇到了暂时的技术问题，我们正在处理。

已完成的工作：
✅ 上季度销售数据分析
✅ 市场趋势识别

建议下一步：
1. 系统将在 5 分钟后自动重试报告生成
2. 你也可以手动触发报告生成
3. 如需立即查看分析结果，我可以先展示数据分析部分
```

---

## 注意事项

- **不要暴露内部错误信息**——用户不需要知道是 API 限流还是智能体崩溃
- **提供可操作的建议**——不要只说"出了问题"，要说"你可以做什么"
- **保持一致性**——降级响应的风格应该和正常响应一致
- **记录降级事件**——降级响应应该被记录到监控系统，用于后续分析
