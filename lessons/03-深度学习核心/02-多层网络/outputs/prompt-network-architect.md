# 网络架构设计提示词

> 当你需要为一个新任务设计神经网络架构时，把这个提示词发给 Claude 或 ChatGPT。

---

## 提示词正文

我正在解决一个机器学习任务，需要你帮我设计神经网络架构。以下是任务信息：

**任务类型：** [分类 / 回归 / 序列预测 / 其他]
**输入数据：** [描述输入的形状、类型、取值范围]
**输出要求：** [描述输出的形状、类别数、取值范围]
**数据规模：** [训练样本数量]
**部署环境：** [服务器 GPU / 移动端 / 浏览器 / 嵌入式]

请帮我设计网络架构，并回答以下问题：

1. **网络深度**：推荐几层隐藏层？为什么？
2. **每层宽度**：每层多少个神经元？为什么选择这个数字？
3. **激活函数**：每层用什么激活函数？ReLU、GELU、Sigmoid 还是其他？
4. **参数量估算**：总共有多少可训练参数？是否在合理范围内？
5. **潜在风险**：这个架构可能遇到什么问题？（过拟合、梯度消失、计算量过大等）
6. **替代方案**：有没有更合适的架构？（CNN、Transformer、RNN 等）

请用具体数字给出推荐架构（如 784-256-128-10），并解释每个设计决策的理由。

---

## 使用示例

**输入：**

```
任务类型：分类
输入数据：28×28 灰度手写数字图像，像素值 0-255
输出要求：10 个类别（数字 0-9）
数据规模：60,000 训练样本
部署环境：服务器 GPU
```

**期望输出：**

```
推荐架构：784-256-128-10

1. 网络深度：2 层隐藏层。对于 MNIST 级别的简单图像任务，2 层足够。
2. 每层宽度：256 → 128，逐层递减。这是经典的漏斗结构，逐层压缩特征。
3. 激活函数：隐藏层用 ReLU，输出层用 Softmax。
4. 参数量：784×256+256 + 256×128+128 + 128×10+10 = 235,146 参数。
5. 潜在风险：全连接网络对空间结构不敏感，如果精度不够考虑 CNN。
6. 替代方案：LeNet-5（2 个卷积层 + 3 个全连接层）是更经典的选择。
```
