# 主题建模——LDA 与 BERTopic

> LDA：文档是主题的混合，主题是词的分布。BERTopic：文档在嵌入空间聚类，每个簇是一个主题。同一个目标，不同的分解方式。

**类型：** 概念课 | **语言：** Python | **前置知识：** 阶段 05 · 02 + 05 · 03 | **预计时间：** ~45 分钟 | **所处阶段：** Tier 1

## 1. 问题

你有 1 万条客服工单、5 万篇新闻、20 万条微博。需要知道这些内容在谈什么——但没有标注类别，甚至不知道有几个类别。

主题建模在无监督条件下回答这个问题。喂给它语料，回道少量连贯主题 + 每篇文档的主题分布。

两套算法主导这个领域。**LDA（2003）**：每个文档是潜主题的混合，每个主题是词的分布。推断是贝叶斯的。2026 年仍在需要混合成员主题分配和可解释词级概率分布的生产场景中运行。**BERTopic（2020）**：用 BERT 编码→UMAP 降维→HDBSCAN 聚类→基于类 TF-IDF 提取主题词。在短文本、社交媒体和语义相似比词重叠更重要的一切场景中胜出。

---

## 2. 概念

### LDA 的生成故事

1. 每个主题 k → 词分布 φ_k ~ Dirichlet(β)
2. 每个文档 d → 主题分布 θ_d ~ Dirichlet(α)
3. 为文档 d 的每个位置：从 θ_d 采样主题 z，从 φ_z 采样词 w

推断逆转这个过程——给定观察到的词，推断每篇文档的主题分布和每个主题的词分布。Collapsed Gibbs 采样：积分掉 φ 和 θ，只采样每词的主题分配 z。

### α 和 β 的直觉

| 参数 | 小值 | 大值 |
|---|---|---|
| α（文档→主题稀疏度） | 每篇文档只有 1-2 个主题 | 每篇文档混合多个主题 |
| β（主题→词稀疏度） | 每个主题只有少量核心词 | 每个主题的词分布更平坦 |

---

## 3. 从零实现

核心采样公式——每个词的候选主题得分：

```
P(z_i=k | ...) ∝ (文档d中主题k的计数 + α) × (主题k中词w的计数 + β) / (主题k的总词数 + V·β)
```

前一项 = 这个文档有多喜欢主题 k。后一项 = 主题 k 有多喜欢这个词。两者相乘——在当前文档和当前词上都"适合"的主题获得高概率。

完整代码包括：随机初始化→构建计数矩阵→Gibbs 迭代（每次扣掉当前词的计数→重算所有候选主题的概率→采样→加回新主题的计数）→输出文档-主题分布和主题-top 词。

代码见 `code/lda_demo.py`。

### LDA vs BERTopic 选择

| 场景 | 选择 |
|---|---|
| 长文本、需要混合成员 | LDA |
| 短文本、社交媒体、语义优先 | BERTopic |
| 可解释的词分布、合规需求 | LDA（概率分布 → 可审计） |
| 未知主题数 | BERTopic（HDBSCAN 自动确定簇数） |

---

## 📚 小结

LDA 用贝叶斯推断将文档分解为潜主题的组合——α 控制文档主题混合的稀疏度，β 控制主题词分布的稀疏度。BERTopic 用嵌入+聚类在语义空间中发现主题。一个适合长文本和混合成员，一个适合短文本和语义聚类。
