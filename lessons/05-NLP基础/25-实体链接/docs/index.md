# 实体链接与消歧

> NER 找到了"Jordan"。实体链接决定：迈克尔·乔丹（篮球）？Michael B. Jordan（演员）？Michael I. Jordan（伯克利 ML 教授）？还是约旦（国家）？没有链接，你的知识图谱永远是模糊的。

**类型：** 实现课 | **语言：** Python | **前置知识：** 阶段 05 · 06（NER）、05 · 24（指代消解） | **预计时间：** ~60 分钟 | **所处阶段：** Tier 1

---

## 🎯 学习目标

- [ ] 理解实体链接的两个子任务——候选生成 + 候选排序
- [ ] 使用 Wikipedia/Wikidata 作为知识库，消歧 NER 找到的实体提及

---

## 1. 问题

"Jordan 击败了媒体。"NER 标注"Jordan"为 PERSON。但**哪个** Jordan？

- 迈克尔·乔丹（篮球）？
- Michael B. Jordan（演员）？
- Michael I. Jordan（伯克利 ML 教授——在 ML 论文中这个混淆真实存在）？
- 约旦（国家名——Jordan 在英文中也可以是国家）？

**实体链接（Entity Linking, EL）**将每个提及消歧为一个知识库中的唯一条目：Wikidata、Wikipedia、DBpedia、或你的领域 KB。

---

## 2. 两阶段流水线

```
NER 提及 "Paris" → [候选生成] → {Paris(France), Paris Hilton, Paris(Texas), Paris(Trojan)}
                              → [候选排序] → Paris(France) Q123456
```

- **候选生成：** 从 KB 中找到所有可能匹配的实体。基于名称匹配（精确/模糊）+ 别名词典 + 首字母缩写扩展
- **候选排序：** 对每个候选打分。三个信号：**（1）上下文相似度**——文档和实体描述之间的向量相似度。**（2）先验概率**——"Paris"在 Wikipedia 中链接到法国首都的次数远多于 Paris Hilton。**（3）全局一致性**——同一文档中的所有实体应该来自一个一致的世界（如果文档已经提到"法国"和"欧洲"，Paris 更可能是法国首都）

---

## 3. 中文实体链接的特殊挑战

- **中文名字的歧义度极高。** "张伟"在全国人口中重名超过 30 万。消歧几乎完全依赖上下文——同篇文档中提到的其他实体（公司、地点、时间）是消歧的唯一线索
- **百度百科 vs Wikipedia。** Wikipedia 中文版条目数远少于英文版——大量中国本土实体（企业家、地方景点、网络红人）只在百度百科中有条目。中文 EL 生产系统通常需要 Wikipedia + 百度百科 + 领域 KB 的联合候选库
- **中英文混用的提及。** 一篇中文新闻中可能同时出现"苹果公司"、"Apple Inc."、"AAPL"——三者指代同一实体。实体链接系统需要跨语言别名映射
