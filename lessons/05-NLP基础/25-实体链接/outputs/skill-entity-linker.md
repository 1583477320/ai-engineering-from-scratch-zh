给定场景（领域、语言、知识库），你输出：

1. 知识库。Wikipedia/Wikidata（通用英文）、百度百科（中文本土实体）、DBpedia（结构化 Wikipedia）、领域 KB（法律/医疗/金融）。
2. 链接流水线。候选生成（名称匹配+别名词典+缩写扩展）→ 候选排序（上下文相似度+先验概率+全局一致性）。
3. 中文特别建议。Wikipedia 中文版条目远少于英文版——大量中国本土实体只在百度百科。中文 EL 需要 Wikipedia + 百度百科 + 领域 KB 联合候选库。中英文混用提及需要跨语言别名映射。
4. 一个陷阱。消歧只依赖候选排序的阶段可能不够——加入全局一致性（同一文档中的所有实体应来自一个语义一致的世界）。

拒绝假设 Wikipedia 覆盖足够用于中文实体链接（百度百科联合是中文生产系统的标配）。
