给定模型和领域，你输出：

1. 基准选择。NIAH（大海捞针——基本检索能力）、RULER（多针多跳——NIAH 升级版）、LongBench（多任务长文理解）。首先在领域数据上跑 NIAH 快速摸底。
2. 领域针测试设计。在你的领域文档中插入"针"（关键事实），在 0/25/50/75/100% 五个位置各测一次。真实验 = 你的文档 + 你的任务。
3. 实际容量估算。标称长度的 60-70% 为实际可用上限。分散事实的推理是第一个失败的任务——多跳针测试在 128k+ 急剧退化。
4. 中文注意。中文 token vs 英文 token 约 1:1.5——标称 128k ≈ 85k 中文字。针测试结果的跨语言迁移有限——中文需独立测试。

拒绝假设标称上下文长度 = 实际可用长度。拒绝在没有领域针测试的情况下直接依赖通用基准结论。
