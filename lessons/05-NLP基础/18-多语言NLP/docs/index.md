## 1. 问题

英文有数十亿条标注。乌尔都语有几千条。迈蒂利语几乎没有。任何服务全球用户的 NLP 系统必须在任务特定训练数据不存在的语言长尾上工作。

多语言模型用**一个模型同时在多种语言上训练**来解决这个问题。共享的表示空间让模型将在高资源语言上学到的技能迁移到低资源语言。在英文情感分析上 fine-tune → 推断时直接在乌尔都语上运行 → 得到意外好的情感预测。这就是零样本跨语言迁移——它重塑了 NLP 如何交付给世界。

## 2. 概念

### 三个共享，一个模型

| 层级 | 说明 |
|---|---|
| **共享词表** | SentencePiece 在全部目标语言的文本上训练。同一子词单元表示跨相关语言的同一词素——英文和意大利文的 `anti-` 是同一个 token |
| **共享表示** | 多语言语料上预训练的 Transformer 学到了"语义相似的句子在不同语言中产生相似的隐藏状态"。"cat"、"chat"、"gato" 的嵌入聚集在一起 |
| **共享任务头** | 分类/标注头对所有语言共享——因为它接收的是一个语言中立的表示 |

### 零样本 vs 少样本

- **零样本：** 只在英文上 fine-tune。推断时跑在任何语言上。无需目标语言标注。对与英文类型相近的语言（德语、法语、西班牙语）效果强，对类型差异大的语言（日语、土耳其语、巴斯克语）效果弱
- **少样本：** 加入 100-500 条目标语言标注。分类任务的准确率跳到英文基线的 95-98%。这是多语言 NLP 中投资回报率最高的单杠杆

### 模型选择策略

| 模型 | 语言数 | 适用场景 |
|---|---|---|
| mBERT | 104 | 经典 NLP 任务基线、学术对比 |
| XLM-RoBERTa | 100 | 更好的跨语言表示——分类/NER 的默认选择 |
| NLLB-200 | 200 | 翻译 + 零样本低资源语言对。训练数据最稀缺的语言 |
| mT5 / ByT5 | 101 / 字节级 | 生成任务——摘要、翻译、QA |

## 3. 中文场景特别建议

- **中英文混合任务时 XLM-R 优于 mBERT。** XLM-R 在中文数据上的预训练量远超 mBERT（CC-100 中文部分 vs Wikipedia 中文），中文子词的覆盖率更好
- **不要假设"所有中文都能一起训练"。** 简体中文和繁体中文在共享词表中是不同的子词序列——如果你需要繁简同时支持，确保训练 tokenizer 的语料中两种变体都有足够占比
- **中文 + 小语种 → 选择源语言时"类型相近度"比"标注数据量"更重要。** 在韩文上 fine-tune 然后迁移到日文（同为 SOV 语序）可能比在英文上 fine-tune 然后迁移到日文更好——尽管英文标注数据量远大于韩文
