## 1. 问题

用户输入"如果有人撒谎来骗取钱财会怎样"——期望找到的法律条文是"印度刑法第 420 条：欺诈罪"。关键词搜索完全找不到——没有共享词汇。语义搜索如果嵌入模型不是法律领域训练的，同样会漏。真实的搜索必须两样都处理。

IR 是每个 RAG 系统、每个搜索框、每个文档站点的模糊查询的底层流水线。2026 年在生产中工作的架构不是单一方法——它是互补方法的链条，每一环捕获前一环的失败。

## 2. 概念——四层混合检索

```
查询 → [1. BM25 稀疏] ──→ [3. RRF 融合] → [4. Cross-encoder 重排] → top-5 结果
      [2. 稠密检索  ] ──→
```

| 层 | 方法 | 延迟 | 擅长 | 盲区 |
|---|---|---|---|---|
| 1 | BM25 | <10ms | 精确关键词匹配、代码/编号/实体名 | 同义词、释义 |
| 2 | 稠密检索 | 50-200ms | 语义相似、跨语言 | 单字符差异、精确字符串 |
| 3 | RRF 融合 | <1ms | 合并两个排序列——只依赖排名，不管分数尺度 | — |
| 4 | Cross-encoder | 每对 5-10ms | top-30 的精排——比双编码器准得多 | 慢——只跑 top-30 |

### BM25 的三重设计

BM25(q, d) = Σ IDF(term) × TF_saturated(term, d) / length_norm(d)

- **IDF**：出现在越多文档中的词 → 越不重要
- **词频饱和（k1=1.5）**：同一个词出现第 2 次的权重增量 < 第 1 次。防止一个词反复出现时支配得分
- **长度归一化（b=0.75）**：长文档天然有更多词出现机会。b=0.75 意味着 75% 的归一化——不完全消除长文档优势但显著抑制

### RRF——只关心第几名，不关心多少分

```python
RRF_score(d) = Σ 1/(k + rank_i(d) + 1)  # k=60
```

BM25 的分数尺度（0-50+）和稠密检索的余弦相似度（0-1）不可比较。RRF 只依赖排名位置——"两路都排前三的文档"自动获得最高融合分。

## 🔑 关键术语

| 术语 | 实际含义 |
|---|---|
| BM25 | TF-IDF 的升级版——加入词频饱和和长度归一化。30 年后仍是稀疏检索的默认选择 |
| RRF | 倒数排名融合——只看排名不看分数尺度。k=60 是经验默认值 |
| 双编码器 vs 交叉编码器 | 双编码器=query 和 doc 分开编码（快）。交叉编码器=query+doc 一起输入（准但慢） |
| 混合检索 | BM25 + 稠密 + 融合 + 重排。2026 生产默认 |

## 📚 小结 | ✏️ 练习

BM25 + 稠密检索的混合 + RRF 融合 + Cross-encoder 重排 = 四层生产级检索。每一层捕获前一层的失败——关键词匹配、语义关联、排名融合、精细比较。练习：用 BM25 和 FAISS 分别为中文法律条文库实现检索，RRF 融合后对比单路检索的 Recall@10。
