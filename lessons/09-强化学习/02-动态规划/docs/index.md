# 动态规划

> 动态规划是 RL 的"完美世界"——假设模型完全已知（状态转移、奖励），直接求解贝尔曼方程。

**类型：** 实现课 | **语言：** Python | **前置知识：** 阶段 09 · 01（MDP）| **时间：** ~60 分钟

---

## 🎯 学习目标

- [ ] 实现策略迭代——交替评估当前策略、改进策略直到收敛
- [ ] 实现价值迭代——直接迭代贝尔曼最优方程
- [ ] 比较两种方法的收敛速度和适用场景

---

## 1. 问题

贝尔曼方程是递归的固定点方程——如果已知转移概率和奖励函数，可以直接求解。两种方法：

**策略迭代：** 先用当前策略计算 V^π(s)，然后用改进的策略（对每个状态选使 V^π 最大的动作）更新。

**价值迭代：** 直接迭代贝尔曼最优方程——每步更新 V*(s) = max_a Σ P(s'|s,a)[r + γV*(s')]。

---

## 🔑 关键术语

| 术语 | 含义 |
|---|---|
| 策略迭代 | 交替：评估策略→改进策略→重复直到收敛 |
| 价值迭代 | 直接迭代贝尔曼最优方程——不显式表示策略 |
| 收敛 | V 或 π 不再变化——找到了最优解 |

---

## 📚 小结

动态规划在 MDP 完全已知时直接求解最优策略。策略迭代和价值迭代是两种方法——前者显式维护策略，后者直接优化价值函数。收敛是确定的——但需要完整的模型，这在现实中几乎不可能。

---

## ✏️ 练习

1. 在 4×4 GridWorld 上实现策略迭代——记录收敛步数
2. 实现价值迭代——对比与策略迭代在相同网格上的结果
3. 将折扣因子 γ 从 0.9 变为 0.5——最优策略如何变化？
