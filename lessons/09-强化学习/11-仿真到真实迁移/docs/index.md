## 1. 问题

在仿真中训练机器人走路很容易——物理引擎完美模拟摩擦、电机响应、传感器。但真实世界有不完美的摩擦、传感器噪声、电机延迟。在真实世界训练太危险（机器人会摔坏）且太慢（没有并行化）。

**现实差距（Reality Gap）**——仿真和真实之间的分布差异——是机器人 RL 的核心问题。你需要一个**对仿真-真实分布偏移鲁棒**的策略。

三个历史方法：(1) 随机化仿真器（域随机化）；(2) 用少量真实数据微调（域适应）；(3) 估计真实系统参数并匹配（系统辨识）。2026 年的主导配方将三者与大规模并行仿真（Isaac Lab、MuJoCo MJX）结合。

## 2. 概念

### 2.1 现实差距的来源

| 来源 | 描述 | 影响 |
|------|------|------|
| 接触力学 | 摩擦系数、接触模型不匹配 | 运动不稳定 |
| 执行延迟 | 电机响应比仿真慢 | 控制信号滞后 |
| 传感器噪声 | IMU、激光雷达的噪声 | 状态估计偏差 |
| 视觉差异 | 光照、纹理、镜头畸变 | 视觉策略失效 |
| 模型遗漏 | 未建模的柔性、关节间隙 | 意外动力学 |

### 2.2 域随机化（Domain Randomization）

Tobin 等人 2017，Peng 等人 2018。在训练期间随机化仿真器中所有可能不同的参数：质量、摩擦系数、电机增益、传感器噪声、光照等。策略学会在"今天是哪种仿真"的条件分布上泛化。如果真实机器人落在训练包络内，策略有效。

**优点：** 不需要真实数据。一个配方适用多个机器人。
**缺点：** 过度随机化产生保守的策略。

### 2.3 系统辨识（SI）

在训练前将仿真器参数拟合到真实数据。精确但无法处理未建模效应。

### 2.4 教师-学生蒸馏

Miki 等人 2022（ANYmal 四足）。训练一个有特权信息（地面真值摩擦、地形高度）的**教师**。蒸馏一个只能看到真实传感器观测的**学生**。学生从历史中推断特权特征。

### 2.5 2026 年的生产配方（四足行走示例）

```
1. 大规模并行仿真 + 域随机化（重力、摩擦、电机增益、负载）
2. 有特权信息的教师策略（地形图、身体速度真值）
3. 蒸馏学生策略——只用本体感知（腿部关节编码器）
4. 可选：真实 IMU 上的观测自适应
5. 部署——零样本到 10+ 环境
```

### 2.6 大规模并行仿真

2024-2026：Isaac Lab、MuJoCo MJX、Brax 在单 GPU 上运行数千并行机器人。PPO + 4096 并行人形机器人在数小时内收集数年经验。

## 5. LLM 视角

### 5.1 Sim2Real 与 LLM 训练的类比

| Sim2Real 概念 | LLM 对应 |
|--------------|---------|
| 仿真器 | 合成训练数据 |
| 真实世界 | 真实用户交互 |
| 现实差距 | 合成数据 vs 真实数据的分布偏移 |
| 域随机化 | 数据增强 |
| 教师-学生蒸馏 | 知识蒸馏（如 R1 → 小模型） |

DeepSeek-R1 的蒸馏版本（Qwen-1.5B 到 Llama-70B）就是 RLHF 的"教师-学生"模式——用 R1 的推理轨迹监督微调小模型。

### 5.2 使用 ChatGPT / Claude 时的直接体验

LLM 面对的"现实差距"是：训练在互联网文本上，部署时面对用户的具体问题。SFT + RLHF 本质上就是"Sim2Real"——从"仿真"（预训练数据分布）到"真实"（用户交互分布）的迁移。域随机化对应数据多样性训练。

## 6. 工程最佳实践

### 6.1 域随机化调优

| 参数 | 推荐范围 | 说明 |
|------|---------|------|
| 摩擦系数 | 真实值 ±50% | 过窄无法泛化，过宽策略过于保守 |
| 电机增益 | 真实值 ±20% | 精确度要求高 |
| 传感器噪声 | σ = 测量噪声的 2-5 倍 | 略微放大 |

### 6.2 ADR（自动域随机化）

开始时随机化范围小，随着策略改进逐步扩大——类似课程学习。

### 6.3 踩坑经验

- **过度随机化**：训练 slip ∈ [0, 0.9]，策略过于保守不敢行动
- **随机化错误的参数**：随机化镜头色相而真实差距在电机延迟——DR 无效
- **无安全盾**：部署时没有低级控制器限速/限扭矩→硬件损坏

## 7. 常见错误

### 错误 1：在单一仿真参数上训练

**现象：** 训练时 slip=0 完美，部署到真实环境（slip>0）立刻失败。

**修复：** 每回合从分布中随机采样参数。

### 错误 2：训练范围太窄

**现象：** 在训练范围内好，但超出范围急剧下降。

**修复：** 使用 ADR（自动域随机化）逐步扩大范围。

## 8. 面试考点

### Q1：域随机化为什么能帮助泛化？（难度：⭐⭐）

**参考答案：**
域随机化通过在训练时暴露策略于多种仿真参数组合，使得策略学到的策略对参数变化鲁棒。类比机器学习中的数据增强——多样性训练提高泛化能力。当真实机器人参数落在训练分布的支撑内时，策略有效。关键是随机化范围要覆盖真实变异，但不能过宽导致策略过于保守。

### Q2：教师-学生蒸馏解决了什么问题？（难度：⭐⭐⭐）

**参考答案：**
教师在仿真中训练时可以用特权信息（地面真值地形、IMU 漂移），但部署时学生只能看到真实传感器数据（没有地面真值）。教师-学生蒸馏让教师教学生从有限的历史观测中推断特权特征——学生学到"从本体感知中推断地面是否粗糙"的能力，这是单用传感器数据无法直接训练的。

## 📚 小结

Sim2Real 是机器人 RL 的核心挑战——仿真完美但真实不完美。域随机化是最简单的基线——随机化仿真参数使策略对变化鲁棒。教师-学生蒸馏更可靠——教师用特权信息，学生从观测推断。2026 年的生产配方：Isaac Lab + DR + 教师-学生蒸馏 + 安全盾。大规模并行仿真使训练接近实时，域随机化几乎无成本。
