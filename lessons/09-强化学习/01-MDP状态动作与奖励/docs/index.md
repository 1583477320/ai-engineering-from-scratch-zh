## 1. 问题

你在写一个象棋机器人、一个库存规划器、一个交易代理、或一个训练推理模型的 PPO 循环。四个不同的领域，一个惊人的事实：**四个都坍缩到同一个数学对象。**

监督学习给你 (x, y) 对，要求你拟合一个函数。强化学习没有标签——只有一个状态流、你采取的动作、以及一个标量奖励。移动赢了吗？库存决策省钱了吗？交易赚钱了吗？LLM 刚生成的 token 让 judge 给了更高奖励吗？

你无法从这个流中学习，直到你把它形式化。"我看到了什么"、"我做了什么"、"接下来发生了什么"、"那有多好"——每个都必须变成一个你可以推理的对象。这个形式化就是马尔可夫决策过程。每一个 RL 算法，包括 RLHF 和 GRPO，都在这个形状上优化。

## 2. 概念

### 2.1 直观理解

想象你在一个迷宫中。你不知道迷宫的完整地图，但你知道：
- **状态**：你当前在哪一个格子里
- **动作**：你可以向上、下、左、右走
- **奖励**：每走一步扣 1 分（鼓励走最短路），到达出口加 100 分
- **折扣**：未来奖励的权重随时间衰减——今天的 100 分比 100 步后的 100 分更有价值

```
迷宫格子（状态）
   ↓
动作选择（上/下/左/右）
   ↓
奖励（走一步扣分，到终点加分）
   ↓
下一个状态（移到新格子）
   ↓
重复...
```

### 2.2 五个对象

| 对象 | 符号 | 含义 | GridWorld 示例 | LLM RLHF 示例 |
|------|------|------|----------------|--------------|
| 状态 | `S` | 代理需要的所有信息 | 格子坐标 | 上下文词元 + 记忆 |
| 动作 | `A` | 代理的可选动作 | 上/下/左/右 | 生成下一个 token |
| 转移 | `P(s'|s,a)` | 给定状态和动作，下一个状态的分布 | 确定性（走一步到新格子） | 确定性（token 已选定） |
| 奖励 | `R(s,a,s')` | 标量信号 | 每步 -1，终点 +100 | judge 给的奖励分 |
| 折扣 | `γ∈[0,1)` | 未来奖励权重 | γ=0.99（~100 步视野） | γ=1（回合短且有界） |

### 2.3 马尔可夫性质

$$P(s_{t+1}|s_t,a_t) = P(s_{t+1}|s_0,a_0,\dots,s_t,a_t)$$

**未来只依赖现在。** 如果不满足——状态表示不完整，不是方法的失败，是状态的失败。

**修复方法：** 如果需要过去 3 帧才能决定——状态不只是当前观测。修复：堆帧（DQN 在 Atari 上堆 4 帧）或使用循环状态（LSTM/GRU 对观测历史编码）。

### 2.4 策略与回报

- **策略** `π(a|s)`：状态→动作概率分布
- **回报** `G_t = r_t + γr_{t+1} + γ^2 r_{t+2} + …`：折扣后的未来奖励总和
- **价值函数** `V^π(s) = E[G_t|s_t=s]`：从 s 开始在策略 π 下的预期回报
- **Q 值** `Q^π(s,a) = E[G_t|s_t=s, a_t=a)`：从 s 采取 a 的预期回报

**每个 RL 算法都估算这两个值之一，然后相应地改进 π。**

### 2.5 贝尔曼方程

$$V^π(s) = \sum_a π(a|s) \sum_{s',r} P(s',r|s,a)[r + γV^π(s')]$$

$$Q^π(s,a) = \sum_{s',r} P(s',r|s,a)[r + γ\sum_{a'} π(a'|s')Q^π(s',a')]$$

将预期回报分解为"这一步的奖励"+"折扣后的未来价值"。递归结构。本阶段的每一个算法都在迭代这个方程。

**折扣因子 γ 的物理意义：**

| γ | 有效视野 `≈ 1/(1-γ)` | 适用场景 |
|---|---------------------|---------|
| 0.9 | 10 步 | 短期决策 |
| 0.95 | 20 步 | 控制任务 |
| 0.99 | 100 步 | 一般控制 |
| 0.999 | 1000 步 | 长期策略游戏 |
| 1.0 | 无限（需要有限回合） | LLM RLHF（回合有限） |

太低→近视；太高→信用分配噪音多。LLM RLHF 通常用 γ=1 因为回合短且有界。控制任务用 0.95-0.99。

## 3. 从零实现

### Step 1：4×4 GridWorld MDP

```python
GRID = 4
TERMINAL = (3, 3)
ACTIONS = {"up": (-1, 0), "down": (1, 0), "left": (0, -1), "right": (0, 1)}

def step(state, action):
    """执行动作，返回 (下一个状态, 奖励, 是否结束)。"""
    if state == TERMINAL:
        return state, 0.0, True
    dr, dc = ACTIONS[action]
    new_r = min(GRID - 1, max(0, state[0] + dr))
    new_c = min(GRID - 1, max(0, state[1] + dc))
    new_state = (new_r, new_c)
    return new_state, -1.0, new_state == TERMINAL
```

五行代码就是整个环境。确定性转移、常数步惩罚、吸收终止状态。

### Step 2：策略评估

```python
def evaluate_policy(policy, gamma=0.99, n_iter=100):
    """迭代贝尔曼方程计算 V^π(s)。"""
    V = [[0.0] * GRID for _ in range(GRID)]
    for _ in range(n_iter):
        V_new = [[0.0] * GRID for _ in range(GRID)]
        for r in range(GRID):
            for c in range(GRID):
                s = (r, c)
                if s == TERMINAL:
                    V_new[r][c] = 0.0
                    continue
                for a in ACTIONS:
                    s_next, reward, _ = step(s, a)
                    V_new[r][c] += policy(s, a) * (reward + gamma * V[s_next[0]][s_next[1]])
        V = V_new
    return V
```

### Step 3：可视化

```python
def print_values(V):
    """将价值函数打印为 4×4 网格。"""
    for r in range(GRID):
        row = [f"{V[r][c]:6.2f}" for c in range(GRID)]
        print("  ".join(row))

# 均匀随机策略下的 V
def uniform_policy(s, a):
    return 0.25  # 四个动作概率均等

V = evaluate_policy(uniform_policy, gamma=0.99)
print_values(V)
# (0,0) 约 -60 到 -80，最优值 -6（最短路径）
```

## 5. LLM 视角

### 5.1 MDP 在大语言模型中的体现

| MDP 元素 | LLM 场景 |
|---------|---------|
| 状态 S | 上下文窗口中的所有词元 + 历史信息 |
| 动作 A | 词表中的每个词元（32K-128K 个动作） |
| 转移 P | 几乎确定性（模型采样一个 token 后状态更新） |
| 奖励 R | 通常只在回合结束时给（judge 打分） |
| 折扣 γ | γ=1（回合短，~200 个 token） |

### 5.2 RLHF 如何利用 MDP

RLHF（基于人类反馈的强化学习）本质上是在一个特殊的 MDP 上训练：
- **状态** = prompt + 部分回复
- **动作** = 下一个 token
- **奖励** = 回复完整后，奖励模型给出的分数
- **折扣** = γ=1（只关心最终奖励）

这个 MDP 的特殊之处：(1) 动作空间极大（~10 万 token）；(2) 回合短（~100-1000 步）；(3) 奖励极其稀疏（通常只在最后一步给分）。

### 5.3 GRPO——更简单的 RL 训练方式

GRPO（Group Relative Policy Optimization）跳过了传统的奖励模型：它用**验证器**（rule-based verifier）直接给答案打 0/1 分。奖励不再是一个需要训练的模型，而是一个简单的函数：答对了=1，答错了=0。

### 5.4 使用 ChatGPT / Claude 时的直接体验

当你让 Claude 生成一个长回复时，它每一步都在做 MDP 中的"动作选择"——预测下一个 token。RLHF 训练告诉它：什么样的 token 序列能获得更高的人类偏好奖励。这就是为什么 Claude 的回复通常比纯 GPT-4 更"有用"——因为它在 MDP 框架下针对人类偏好做了优化。

## 6. 工程最佳实践

### 6.1 MDP 建模清单

在写任何训练循环之前，先填写这个表格：

| 字段 | 你的场景 | 注意事项 |
|------|---------|---------|
| 状态 | 具体的特征向量或张量规格 | 证明马尔可夫性 |
| 动作 | 离散集合或连续范围 | 维度 |
| 转移 | 确定性 / 已知模型 / 仅可采样 | 决定了用什么算法 |
| 奖励 | 函数和来源 | 稀疏 vs 密集；回合级 vs 步级 |
| 折扣 γ | 值和视野理由 | γ=1 需要有限回合 |

### 6.2 中文场景特别建议

- 在国内 RL 环境中，Minigrid 和 Gymnasium 可通过国内镜像安装：`pip install gymnasium -i https://pypi.tuna.tsinghua.edu.cn/simple`
- 中文 LLM RLHF 研究（如通义千问、文心一言）使用了类似 MDP 的训练框架

### 6.3 踩坑经验

- **奖励尺度错误**：奖励 {+100, -100} 和 {+1, -1} 给出相同的最优策略，但梯度幅度差 100 倍。在 PPO/DQN 之前归一化到 [-1, 1]
- **状态非马尔可夫**：如果需要过去 3 个观测才能决策——状态表示不完整，修复方法是堆帧或使用 LSTM
- **γ=1 在无限回合任务上**：每个状态的价值都是无穷大。要么有限回合，要么 γ<1

## 7. 常见错误

### 错误 1：忽略终止状态的 Bellman 处理

**现象：** 终止状态的价值不断更新，导致 V 不收敛。

**原因：** 如果对终止状态应用 Bellman 方程，它仍会"选择最佳动作"——但终止状态之后没有状态了。

**修复：**

```python
# ❌ 错误：对终止状态也更新
for s in all_states():
    v = max(Q[s][a] for a in ACTIONS)  # 终止状态也在计算！

# ✓ 正确：终止状态价值固定为 0
for s in all_states():
    if s == TERMINAL:
        V[s] = 0.0
        continue
    v = max(Q[s][a] for a in ACTIONS)
```

### 错误 2：用平均误差而非最大误差判断收敛

**现象：** 策略看起来稳定了，但实际没有收敛到最优解。

**原因：** 平均误差可以很小但某些状态误差很大——理论保证是 sup-范数（最大误差）。

**修复：**

```python
# ❌ 错误：平均误差
delta = sum(abs(V_new[s] - V[s]) for s in states()) / len(states())

# ✓ 正确：sup-范数
delta = max(abs(V_new[s] - V[s]) for s in states())
```

### 错误 3：状态空间爆炸时不使用函数近似

**现象：** 动态规划在超过 ~10⁷ 个状态时内存不足或速度极慢。

**原因：** DP 是 O(|S|·|A|) 的——状态空间指数增长。

**修复：**

```python
# ❌ 错误：表格方法处理大状态空间
V = {}  # 每个状态一个条目——10⁷ 条目 = 内存爆炸

# ✓ 正确：使用函数近似（神经网络）
import torch.nn as nn
V_network = nn.Linear(state_dim, 1)  # 参数量与状态空间无关
```

## 8. 面试考点

### Q1：马尔可夫性质为什么重要？如果状态不满足马尔可夫性会怎样？（难度：⭐⭐）

**参考答案：**
马尔可夫性质保证了"给定当前状态，未来与过去独立"——这使得贝尔曼方程成立，使得动态规划、TD 学习、Q-learning 等算法可以正确工作。如果状态不满足马尔可夫性（例如只用当前帧作为状态，不包含运动信息），Bellman 方程的前提不成立——估计出的价值函数会不一致，学习过程无法收敛。修复方法：(1) 堆帧——将过去 4 帧堆叠为一个状态；(2) 循环编码——用 LSTM/GRU 将历史编码为向量；(3) 扩展状态——加入速度、加速度等隐藏信息。

### Q2：折扣因子 γ 从 0.99 改为 0.5 会怎样？（难度：⭐⭐）

**参考答案：**
γ=0.99 的有效视野约 100 步——代理会考虑 100 步后的奖励，倾向于做短期牺牲换取长期回报。γ=0.5 的有效视野约 2 步——代理几乎只关心下一步的奖励。在 GridWorld 中，γ=0.99 时代理可能绕远路寻找更快到达终点的路径；γ=0.5 时代理会选择当前奖励最高（步数最少）的动作，可能被困在局部最优。在实际应用中，γ 的选择取决于任务的时间尺度——控制任务通常 0.95-0.99，短期决策可能用更低的 γ。

### Q3：为什么需要折扣因子 γ？为什么不用 γ=1？（难度：⭐⭐⭐）

**参考答案：**
理论上，如果回合有限（如象棋，一定会有胜负），γ=1 是可以的——回报是有限和。但在无限回合任务（如持续控制）中，γ=1 会导致回报无穷大，贝尔曼方程没有有限的解。γ<1 的数学保证：贝尔曼算子是 sup-范数下的 γ-压缩映射——压缩映射保证了唯一不动点和几何收敛速度。没有 γ<1，收敛性不保证。实际中 γ 还有一个直觉意义：它编码了代理对"延迟满足"的偏好——γ 越高，代理越愿意接受短期代价换取长期回报。

## 🔑 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|----------|---------|
| MDP | "强化学习的设定" | 元组 `(S, A, P, R, γ)` 满足马尔可夫性质——RL 的数学基础 |
| 状态 | "代理看到的东西" | 策略类下未来动态的充分统计量——不是"看到了什么"，而是"需要知道什么" |
| 策略 | "代理的行为" | 条件分布 `π(a|s)` 或确定性映射 s→a |
| 回报 G_t | "总奖励" | 从当前步起的折扣奖励总和 `Σγ^t r_t` |
| 价值 V^π(s) | "一个状态有多好" | 在策略 π 下从 s 开始的预期回报 |
| Q 值 Q^π(s,a) | "一个动作有多好" | 在策略 π 下从 s 采取 a 的预期回报 |
| 贝尔曼方程 | "动态规划递归" | 价值 / Q 的一步分解：当前奖励 + 折扣后继价值 |
| 折扣 γ | "未来 vs 现在" | 远期奖励的几何衰减权重；有效视野 `≈ 1/(1-γ)` |

## 📚 小结

MDP = 状态 + 动作 + 转移 + 奖励 + 折扣。贝尔曼方程将回报分解为递归结构——所有 RL 算法都在迭代这个方程。本阶段的所有算法——Q-learning、PPO、DPO、GRPO——都在这个形状上优化。学一次 MDP，后面免费读。下一课我们将用动态规划直接求解贝尔曼方程——假设模型完全已知时的最优策略。
