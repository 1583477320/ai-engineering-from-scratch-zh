# 演员-评论家——A2C 与 A3C

> REINFORCE 方差大。DQN 不能处理连续动作。演员-评论家结合两者——演员输出策略，评论家评估价值，共同训练。

**类型：** 实现课 | **语言：** Python | **前置知识：** 阶段 09 · 05（DQN）、06（策略梯度）| **时间：** ~75 分钟

---

## 🎯 学习目标

- [ ] 理解演员-评论家架构——演员（策略 π）和评论家（价值 V 或 Q）协同工作
- [ ] 实现 A2C——用优势函数降低策略梯度方差
- [ ] 说明 A3C 的异步并行优势——多个环境同时采样

---

## 1. 问题

DQN 好在稳定但不能处理连续动作空间。REINFORCE 可以处理连续动作但方差太大。演员-评论家（Actor-Critic）结合两者：**演员输出策略（选择动作），评论家评估价值（给演员一个低方差的基线）**。

### A2C vs A3C

| | A2C | A3C |
|---|---|---|
| 训练方式 | 同步：所有 worker 计算梯度后同步更新 | 异步：每个 worker 独立更新主网络 |
| 稳定性 | 更稳定（同步更新） | 更快但可能不稳定 |
| 复杂度 | 简单 | 需要多线程 |

---

## 🔑 关键术语

| 术语 | 含义 |
|---|---|
| 演员（Actor） | 输出策略 π(a|s)——给定状态选择动作的分布 |
| 评论家（Critic） | 评估价值 V(s) 或 Q(s,a)——给演员一个低方差的基线 |
| 优势函数 | A(s,a) = Q(s,a) - V(s)——动作相对于"平均水平"的好坏 |
| A2C | 同步演员-评论家——所有 worker 计算梯度后同步更新 |
| A3C | 异步演员-评论家——每个 worker 独立更新主网络 |

---

## 📚 小结

演员-评论家结合了策略梯度（演员）和价值函数（评论家）。A2C 用同步更新更稳定，A3C 用异步并行更快但更复杂。在现代实践中，PPO（阶段 08）取代了 A2C/A3C——通过 Clipped Surrogate Objective 实现了稳定且高效训练。
