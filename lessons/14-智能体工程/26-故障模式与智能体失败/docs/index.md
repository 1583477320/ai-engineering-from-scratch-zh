## 1. 问题

智能体系统在生产中频繁失败。MASFT（伯克利，2025）系统化了 14 种故障模式——分为规划故障、执行故障、协调故障。微软的分类法补充了 AI 特有的故障。理解这些故障模式是构建可靠智能体的第一步。

---

## 2. 概念

### 2.1 MASFT 三类故障

| 类别 | 具体模式 |
|------|---------|
| **规划故障** | 规划错误、目标偏移、步骤遗漏、过度自信 |
| **执行故障** | 工具调用错误、参数错误、超时、权限不足 |
| **协调故障** | 通信失败、状态不一致、死锁、资源竞争 |

### 2.2 五种常见故障模式

| 故障 | 描述 | 影响 |
|------|------|------|
| **幻觉动作** | 执行了不存在的操作 | 任务失败 |
| **范围蔓延** | 任务范围不断扩大 | 成本失控 |
| **级联错误** | 前一步的错误传播到后续步骤 | 整个任务失败 |
| **上下文丢失** | 长对话中关键信息被遗忘 | 回答不准确 |
| **工具滥用** | 错误地调用工具或参数错误 | 效率低、错误 |

### 2.3 故障检测策略

| 策略 | 方法 |
|------|------|
| **输出验证** | 检查工具返回值是否符合预期 |
| **一致性检查** | 多代理答案一致性 |
| **超时监控** | 检测无限循环或超时 |
| **成本监控** | 检测成本异常增长 |

## 3. 从零实现

### Step 1：故障检测器

```python
class FaultDetector:
    """智能体故障检测器。"""
    def __init__(self):
        self.history = []

    def check_tool_abuse(self, tool_calls, max_per_minute=10):
        """检测工具滥用。"""
        recent = [c for c in tool_calls[-max_per_minute:]]
        if len(recent) > max_per_minute:
            return True, "工具调用频率异常"
        return False, None

    def check_cascading_error(self, step_results):
        """检测级联错误。"""
        error_count = sum(1 for r in step_results if r.get("error"))
        if error_count > len(step_results) * 0.5:
            return True, "超过50%步骤出错——级联错误"
        return False, None

    def check_scope_creep(self, task, completed_steps):
        """检测范围蔓延。"""
        if len(completed_steps) > 20:
            return True, f"已完成{len(completed_steps)}步——可能存在范围蔓延"
        return False, None
```

### Step 2：故障模式分类

```python
FAULT_CATEGORIES = {
    "planning": ["规划错误", "目标偏移", "步骤遗漏", "过度自信"],
    "execution": ["工具调用错误", "参数错误", "超时", "权限不足"],
    "coordination": ["通信失败", "状态不一致", "死锁", "资源竞争"],
}

def classify_fault(error_msg):
    """分类故障模式。"""
    for category, patterns in FAULT_CATEGORIES.items():
        for pattern in patterns:
            if pattern in error_msg:
                return category
    return "unknown"
```

## 5. 工程最佳实践

### 5.1 防御深度

```
1. 输入验证：过滤恶意输入
2. 规划验证：检查计划合理性
3. 执行验证：检查工具返回值
4. 结果验证：检查最终输出
5. 监控告警：异常时自动停止
```

### 5.2 踩坑经验

- **级联错误**：一步出错→后续全部失败→需要回滚机制
- **幻觉动作**：LLM 编造不存在的工具→始终验证工具存在性
- **无限循环**：智能体反复执行相同操作→设置最大步数

---

## 6. 常见错误

### 错误 1：不设置超时和步数限制

**现象：** 智能体陷入无限循环——资源耗尽。

**修复：** 每个工具调用设置超时；智能体设置 max_steps。

### 错误 2：忽略级联错误

**现象：** 一个小错误传播到整个任务——直到最后才发现。

**修复：** 每步检查结果有效性——早期发现早期停止。

## 7. 面试考点

### Q1：智能体最常见的五种故障模式是什么？（难度：⭐⭐）

**参考答案：**
(1) **幻觉动作**——执行不存在的操作；(2) **范围蔓延**——任务范围不断扩大；(3) **级联错误**——前一步错误传播到后续；(4) **上下文丢失**——长对话中关键信息被遗忘；(5) **工具滥用**——错误调用工具或参数。这五种模式覆盖了 80% 的生产智能体故障。
