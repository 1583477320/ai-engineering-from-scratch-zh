# 智能体工作台工程：为什么强大模型仍然失败

> 强大的模型是不够的。可靠的智能体需要一个工作台：指令、状态、范围、反馈、验证、审查和交接。剥离这些，即使是前沿模型也会产生不安全的产品。

## 1. 问题

即使 GPT-4o / Claude Opus / Gemini 3 Pro 这样的前沿模型也会在简单任务上失败——不是因为模型不够聪明，而是因为**没有正确的工作台**。模型像一名优秀但缺乏工具的工匠——在合适的工作台上它创造奇迹，否则也会犯错。

---

## 2. 概念

### 2.1 智能体工作台的七个组件

| 组件 | 说明 | 类比 |
|------|------|------|
| **指令** | 清晰的任务描述 | 文档 |
| **状态** | 当前进度和历史 | 上下文 |
| **范围** | 任务边界——不多不少 | 需求 |
| **反馈** | 执行结果 | 测试 |
| **验证** | 正确性检查 | 审查 |
| **审查** | 人类审核 | 批准 |
| **交接** | 跨智能体协作 | 转交 |

### 2.2 没有工作台时

```
用户输入 → [LLM] → 输出
        ↕             ↕
   无范围限制     无验证机制

问题：模型可能偏离任务、产生幻觉、忽略限制
```

### 2.3 有工作台时

```
用户输入 → [指令 + 状态 + 范围] → [LLM] → [验证 + 反馈] → 输出
                                                  ↕
                                            [审查 + 交接]
```

## 3. 从零实现

### Step 1：工作台组件

```python
class AgentWorkbench:
    """最小的智能体工作台。"""
    def __init__(self, instructions, scope):
        self.instructions = instructions  # 任务描述
        self.state = {}                   # 状态跟踪
        self.scope = scope                # 任务范围
        self.history = []                 # 历史记录

    def validate(self, task):
        """验证任务是否在范围内。"""
        return any(s in task.lower() for s in self.scope)

    def execute(self, task, llm_fn):
        """在工作台范围内执行。"""
        if not self.validate(task):
            return {"error": "超出任务范围"}

        self.history.append({"task": task, "status": "started"})
        result = llm_fn(task, context=self.history)
        self.state[task] = result
        self.history[-1]["status"] = "completed"

        return {"result": result, "status": "success"}
```

## 6. 常见错误

### 错误 1：没有范围限制

系统提示：模型可以访问超出任务范围的内容——偏离任务。

修复：定义明确的范围——模型只能访问任务所需的内容。

## 7. 面试考点

### Q1：为什么强大模型在简单任务上失败？（难度：⭐⭐）

**参考答案：**
不是因为模型不够聪明——而是因为没有工作台。工作台提供指令（做什么）、状态（做到哪儿了）、范围（不要做什么）、反馈（做对了吗）、验证（检查一下）、审查（人来看看）。剥离这些，再强的模型也会偏离方向。

|------|----------|---------|
| 工作台 | "智能体工具台" | 指令+状态+范围+反馈+验证+审查+交接——让模型可靠执行的基础设施 |
| 范围 | "任务边界" | 模型只能访问和执行范围内定义的内容 |

## 📚 小结

模型能力 ≠ 执行可靠性。智能体工作台提供七个组件：指令、状态、范围、反馈、验证、审查、交接。剥离工作台，前沿模型也会产生不安全的产品。
