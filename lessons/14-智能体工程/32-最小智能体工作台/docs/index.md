# 最小智能体工作台

> 最小的有用工作台是三个文件：一个根指令路由、一个状态文件、一个任务板。其他一切都在此之上分层构建。如果一个仓库无法携带这三个文件，没有模型能拯救它。

**类型：** 实现课 | **语言：** Python | **前置知识：** 阶段 14 · 31（为什么强大模型失败）| **时间：** ~45 分钟
**所处阶段：** Tier 3

## 1. 问题

全功能工作台很好——但开始使用太复杂。**三个文件就够了**：指令路由（做什么）、状态文件（做到哪了）、任务板（下一步做什么）。没有这三个文件，任何前沿模型也无法可靠工作。

---

## 2. 概念

### 2.1 三个文件

| 文件 | 用途 | 类比 |
|------|------|------|
| **指令路由** | 任务描述、规则、边界 | 项目 README |
| **状态文件** | 当前进度、已完成任务 | 检查点 |
| **任务板** | 待办事项、优先级 | TODO 列表 |

### 2.2 为什么三个文件就够了

```
指令路由 → "这个项目的规则是什么？"
状态文件 → "目前做到哪一步了？"
任务板 → "下一步应该做什么？"

有了这三个，智能体就能开始工作
```

### 2.3 指令路由示例

```python
INSTRUCTIONS = {
    "role": "你是一个帮助完成任务的分析助手",
    "rules": [
        "只回答用户明确提问的内容",
        "如果不确定，请说明",
        "不要超出任务范围",
    ],
    "tools": ["search", "calculate"],
}
```

## 3. 从零实现

### Step 1：最小工作台

```python
class MinimalWorkbench:
    """最小智能体工作台——三文件。"""
    def __init__(self, instructions_file, state_file, task_board):
        self.instructions = instructions_file
        self.state = state_file
        self.tasks = task_board

    def get_context(self):
        """获取智能体需要的全部上下文。"""
        return {
            "instructions": self.instructions,
            "state": self.state,
            "tasks": self.tasks,
        }

    def update_state(self, task, result):
        """更新状态文件。"""
        self.state[task] = result

    def complete_task(self, task):
        """完成任务。"""
        if task in self.tasks:
            self.tasks.remove(task)
        self.state[f"completed_{task}"] = "done"
        return {"remaining": len(self.tasks), "completed": list(self.state.keys())}


# 三个文件的简化表示
INSTRUCTIONS = {
    "role": "分析助手",
    "rules": ["不超出范围", "不确定时说不知道"],
}
STATE = {}
TASKS = ["分析数据", "生成报告", "审查结果"]
```

## 5. 工程最佳实践

- **三个文件从第一天开始**：即使只有一个任务
- **状态文件自动更新**：每一步后写入
- **任务板从简**：不超过 5 个待办

---

## 6. 常见错误

### 错误 1：太多文件

**现象：** 工作台文件超过 10 个，智能体不知道看哪个。

**修复：** 三个文件就够——指令、状态、任务。
