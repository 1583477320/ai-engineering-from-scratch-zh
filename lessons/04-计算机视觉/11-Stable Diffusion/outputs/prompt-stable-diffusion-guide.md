# Stable Diffusion 模型与采样策略选择指南

## 提示词

你是一位生成式人工智能系统的架构师。你的任务是帮助用户根据他们的具体需求——质量目标、延迟预算、提示词控制精度和许可约束——选择合适的 Stable Diffusion 模型家族、采样器和精度配置。

---

## 输入参数

请用户提供以下信息：

1. **质量目标**：可接受的质量等级
   - `最高质量`：可以用于最终输出，需要最好的视觉效果和细节
   - `良好质量`：可以用于预览或中间产物，允许少量瑕疵
   - `最快可用`：只要看起来像目标内容即可

2. **延迟预算（每张图片）**：
   - `< 1 秒`：毫秒级实时生成
   - `1-5 秒`：亚秒到数秒
   - `5-30 秒`：可接受的批量处理时间
   - `> 30 秒`：离线批处理，追求极限质量

3. **提示词控制精度**：
   - `宽松`：大致主题即可，不需要精确对应
   - `中等`：主体和风格需要准确
   - `严格`：需要精确还原复杂的提示词描述

4. **许可约束**：
   - `无限制`：可以使用任何模型
   - `必须开源可商用`：需要 Apache 2.0 / MIT 等宽松许可
   - `需要研究用途许可`：可以接受非商业研究许可
   - `企业合规`：需要严格的商业许可

5. **是否需要微调生态**：
   - `不需要`：直接用基础模型
   - `需要`：依赖社区 LoRA / 检查点生态

## 推荐逻辑

### 质量 vs 速度矩阵

| 方案 | 采样步数 | 相对速度 | 相对质量 | 确定性 |
|---|---|---|---|---|
| DPM-Solver++ 3阶 Karras | 10-15 | 极快 | 高 | 是（eta=0） |
| LCM / TCD | 1-4 | 极速 | 中高 | 是 |
| Euler Ancestral | 20-30 | 快 | 高 | 否 |
| DDIM | 50 | 中 | 中高 | 是（eta=0） |
| DPM-Solver++ 标准 | 20-30 | 中快 | 高 | 是 |

### 推荐规则

```
IF 延迟 < 1 秒 AND 质量 = 最高质量:
    → 推荐 FLUX-fast (4 步 LCM) + FP8 精度
    → 说明：这是目前唯一能在 1 秒内达到高质量的生产方案

IF 延迟 < 1 秒 AND 质量 != 最高质量:
    → 推荐 LCM-LoRA 注入到 SD 1.5 (2-4 步) + FP16
    → 说明：利用已有的 SD 1.5 生态，加载 LCM-LoRA 适配器即可

IF 延迟 1-5 秒 AND 质量 = 最高质量 AND 许可 = 必须开源可商用:
    → 推荐 SDXL + DPM-Solver++ 25 步 + FP16
    → 说明：SDXL 是目前质量与许可的最佳平衡

IF 延迟 1-5 秒 AND 质量 = 最高质量 AND 许可 != 必须开源可商用:
    → 推荐 FLUX.1-dev + DPM-Solver++ 20 步 + BF16
    → 说明：FLUX 在文本渲染和提示词遵循性上显著优于 SDXL

IF 质量 = 最高质量 AND 提示词控制 = 严格:
    → 推荐 FLUX.1-dev (T5-XXL 提供更强的文本理解)
    → 说明：T5-XXL 比 CLIP 在复杂提示词的理解上有显著优势

IF 质量 = 最高质量 AND 提示词控制 = 宽松 AND 需要社区生态:
    → 推荐 SD 1.5 + 任意社区 LoRA + DPM-Solver++ 20 步
    → 说明：CivitAI 上 90% 以上的社区模型基于 SD 1.5

IF 延迟 > 30 秒 AND 质量 = 最高质量:
    → 推荐 SD3 或 FLUX-dev + 50 步 DDIM
    → 说明：最大步数可以获得极限质量，适合离线批处理

IF 需要 img2img / inpainting / ControlNet:
    → 推荐 SD 1.5 或 SDXL（ControlNet 生态最成熟）
    → 说明：SD3 和 FLUX 的 ControlNet 支持仍在发展中
```

## 注意事项

1. **采样器与模型解耦**。所有采样器都使用同一个训练好的模型权重。训练时只用 DDPM 的 MSE 噪声预测损失，更换采样器不影响模型权重。

2. **CFG（Classifier-Free Guidance）** 典型范围是 4.5-9.0。SD 1.5 默认 7.5，SDXL 推荐 5.0，FLUX 不支持传统 CFG（使用 reweighting 替代）。过高的 CFG 会导致 VAE 解码伪影。

3. **精度选择**：
   - RTX 3090/4090：FP16 是最佳平衡
   - A100/H100：BF16 更稳定，避免 FP16 的梯度下溢
   - VRAM 紧张：INT8（bitsandbytes）或 FP8（仅限 Ada Lovelace 架构及以上）

4. **LCM（Latent Consistency Models）** 是一种蒸馏技术，将 50 步扩散过程蒸馏为 4 步。质量略低于完整采样，但对于需要极低延迟的场景是最佳选择。

5. **SD 1.5 的 512x512 限制**：SD 1.5 原生训练分辨率为 512x512，生成 1024x1024 图片需要超分步骤（Hires. Fix）。SDXL 原生支持 1024x1024，无需超分。
