# Vision Transformer 架构选型指南

> 当你需要在图像分类、目标检测或分割任务中选择视觉模型架构时，使用本提示词分析最佳选择。

---

## 使用方法

将以下提示词发送给 AI 助手，并附上你的任务信息。

## 快速决策树

```
有标注数据吗？
├── 否 → 有大量无标注数据？
│   ├── 是 → DINO/DINOv2 自监督预训练 + 线性探测
│   └── 否 → CNN（ResNet/ConvNeXt），ViT 需要太多数据
└── 是 → 数据量 > 10 万张？
    ├── 是 → 检查推理设备
    │   ├── 边缘/Mobile → ConvNeXt 或 Swin-Ti
    │   ├── 云端 GPU → ViT-B 或 Swin-B
    │   └── 不限 → ViT-H / DINOv2
    └── 否（< 10 万张）→ 检查任务类型
        ├── 检测/分割 → Swin-V2（层次化特征）
        ├── 分类 → ConvNeXt-Ti（小数据更稳）
        └── 多模态 → ViT（CLIP 生态最好）
```

## 常见场景推荐

### 场景 1：手机 App 上的图像分类（1000 类）
**推荐：** ConvNeXt-Ti 或 MobileViT
**理由：** ConvNeXt 对边缘硬件编译友好，推理速度快；MobileViT 是专为移动端设计的轻量 ViT
**预训练：** ImageNet-1k 微调即可

### 场景 2：工业缺陷检测（数据量小，几百到几千张）
**推荐：** ConvNeXt-Small + 强数据增强
**理由：** ViT 在小数据上极易过拟合，ConvNeXt 的归纳偏置是天然的正则化器
**预训练：** ImageNet-21k 或 COCO 预训练的 ConvNeXt

### 场景 3：自动驾驶目标检测（实时性 < 30ms/帧）
**推荐：** RT-DETR 或 Swin-V2  + Deformable DETR Head
**理由：** RT-DETR 是专为实时检测优化的端到端 Transformer；Swin 提供层次化特征利于检测
**预训练：** PASCAL VOC / COCO 预训练 + 自驾场景微调

### 场景 4：医疗影像分割（高分辨率 CT/MRI）
**推荐：** Swin-Unet 或 Swin-V2 + U-Net Decoder
**理由：** Swin 的层次化结构提供多尺度特征，窗口注意力在高分辨率下计算效率高
**预训练：** 用 ImageNet 预训练的 Swin 权重初始化编码器

### 场景 5：多模态学习（图文检索/VLM）
**推荐：** ViT-L/14（CLIP/SigLIP 预训练）
**理由：** CLIP 生态中 ViT 是最成熟的图像编码器，图文嵌入空间对齐最好
**预训练：** openai/clip-vit-large-patch14 或 SigLIP 预训练权重

## 关键选型原则

1. **数据少 → 选 CNN/ConvNeXt**（归纳偏置保护）
2. **数据多 → 选 ViT**（从数据中学习最优表示）
3. **边缘设备 → 选 ConvNeXt**（编译优化好）
4. **高分辨率任务 → 选 Swin**（线性复杂度）
5. **端到端检测/分割 → 选 DETR 系**（无需 NMS/后处理）
6. **多模态对齐 → 选 ViT**（CLIP 生态最成熟）
