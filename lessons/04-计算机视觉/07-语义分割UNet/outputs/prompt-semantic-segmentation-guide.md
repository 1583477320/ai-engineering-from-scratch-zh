# 语义分割任务分析与设计提示词

> 将任何图像分割需求转化为结构化技术方案的结构化提示。

---

## 使用说明

将此模板提供给 LLM，描述你的分割任务，获取结构化的技术方案建议。

---

## 提示词模板

```
你是一位计算机视觉工程师，擅长语义分割和实例分割任务。请根据以下信息分析任务并给出技术方案建议。

【任务描述】
- 任务类型：[语义分割 / 实例分割 / 全景分割]
- 输入数据类型：[RGB照片 / 灰度医学影像 / 卫星遥感 / 红外 / 深度图]
- 输入分辨率：[如 512×512]
- 类别数量：[N]
- 每类样本数：[各类数量的分布]
- 标注质量：[像素级标注 / 多边形标注 / 边界框标注]
- 部署环境：[服务器 GPU / 边缘设备 / 手机端]
- 延迟要求：[__ms 以内]
- 精度要求：[mIoU ≥ __% 或 Dice ≥ __]

【分析要求】

1. **架构选型**：推荐具体的模型架构（如 U-Net、DeepLabV3+、HRNet 等），并给出理由。考虑数据规模、类别数、部署约束。

2. **编码器选择**：如果使用了预训练骨干（如 ResNet、EfficientNet），说明为什么适合该任务。

3. **损失函数设计**：对于每个类别的样本量分布，推荐合适的损失函数组合（如 CE + Dice、Focal Loss、Boundary Loss）。说明理由。

4. **数据增强策略**：针对该数据类型的具体建议。区分几何变换和颜色/强度变换。

5. **评估指标**：除了 mIoU/Dice，还有哪些指标需要考虑？比如小目标的 IoU、边界 F1 分数、轮廓 Hausdorff 距离。

6. **部署优化**：量化方案（FP16 / INT8）、剪枝策略、是否有模型压缩需求。

7. **风险点**：这个任务最可能失败的 3 个原因及缓解方案。

请以工程实践的角度回答，不要泛泛而谈。
```

## 示例

### 输入

```
【任务描述】
- 任务类型：语义分割
- 输入数据类型：卫星遥感 RGB
- 输入分辨率：1024×1024
- 类别数量：5（建筑、植被、水体、道路、裸地）
- 每类样本数：总计约 2000 张图像，各类比例约 20:40:10:20:10
- 标注质量：像素级多边形标注
- 部署环境：服务器（A100 GPU）
- 延迟要求：无实时要求
- 精度要求：mIoU ≥ 0.75
```

### 输出要点（供参考）

1. **架构选型**：DeepLabV3+ ResNet-101。遥感场景需要大感受野（建筑和道路跨度大），ResNet-101 提供更深的层次提取，ASPP 模块可捕获多尺度上下文。
2. **数据增强**：随机裁剪 + 水平/垂直翻转（遥感数据方向无关性）。注意：不可做颜色随机变换（卫星影像的颜色反映真实地表光谱）。
3. **损失函数**：CE + Dice。植被（40%）占据主导，Dice 可以帮助平衡小类（裸地 10%）。
4. **部署优化**：服务器 A100，显存充足，无需压缩。可尝试 Mixed Precision (FP16) 加速训练。

## 快速参考：常见任务的技术栈对照表

| 任务 | 推荐架构 | 损失 | 增强 | 备注 |
|---|---|---|---|---|
| 医学 2D 分割 | U-Net / Modulated U-Net | CE + Dice | 弹性形变、旋转 | 3D 用 3D U-Net |
| 卫星遥感 | DeepLabV3+ / HRNet | CE + Focal | 随机裁剪、翻转 | 不做颜色增强 |
| 自动驾驶 | DeepLabV3+ / CCNet | CE + Dice | 强增强（CutMix、MixUp） | 需实例感知 |
| 工业缺陷检测 | U-Net 轻量化版 | BCE + Dice | 合成缺陷增强 | 负样本远多于正样本 |
| 面部器官分割 | PSPNet / BiSeNet | CE + Dice | 仿射变换 | BiSeNet 适合实时 |
| 细胞分割 | U-Net + watershed 后处理 | CE + Dice + Boundary | 旋转 + 弹性变形 | 后处理是关键 |
