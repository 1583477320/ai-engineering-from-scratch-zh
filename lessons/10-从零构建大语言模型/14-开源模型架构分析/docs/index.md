## 1. 问题

2026 年有数十个开源 LLM 可选。但它们不是随意堆砌的——每个架构选择都有明确的理由。理解这些选择，你就能在不同场景下做出正确的模型选择。

## 2. 概念

### 2.1 2026 年开源 LLM 的"统一架构"

| 组件 | 选择 | 为什么 |
|------|------|--------|
| 归一化 | Pre-LN（Transformer 前） | 训练更稳定，收敛更快 |
| 注意力 | GQA（分组查询注意力） | KV 缓存更小，推理更快 |
| 激活函数 | SwiGLU | 比 ReLU/GELU 更好 |
| 位置编码 | RoPE | 可扩展到任意长度 |
| 归一化层 | RMSNorm（无偏置） | 比 LayerNorm 更快 |

### 2.2 Llama 3（Meta, 2024）

| 参数 | 8B | 70B | 405B |
|------|-----|------|-------|
| 层数 | 32 | 80 | 126 |
| 隐藏维度 | 4096 | 8192 | 16384 |
| 注意力头 | 32 | 64 | 128 |
| KV 头 | 8 | 8 | 8 |
| GQA 比例 | 4:1 | 8:1 | 16:1 |
| 词表大小 | 128,256 | 128,256 | 128,256 |

**关键设计：**
- **GQA（Grouped Query Attention）**：32 个查询头共享 8 个 KV 头——KV 缓存减少 4x
- **128K 词表**：大词表减少序列长度，适合多语言
- **15T token 训练**：比 Llama 2 的 2T 多 7.5 倍

### 2.3 Qwen2.5（阿里, 2024）

| 参数 | 0.5B | 1.5B | 7B | 14B | 72B |
|------|------|------|-----|------|------|
| GQA 配置 | 8kv | 8kv | 8kv | 8kv | 8kv |
| 注意力头 | 14 | 16 | 32 | 40 | 64 |
| 训练 token | 3T | 3T | 18T | 18T | 18T |

**关键设计：**
- **统一 GQA 配置**：所有规模都用 8 KV 头——推理效率一致
- **18T token**：中英文混合训练——中文能力最强
- **上下文长度**：原生支持 32K，扩展到 128K

### 2.4 Mistral（Mistral AI, 2023-2024）

| 特性 | Mistral 7B | Mistral Mixtral 8x7B |
|------|-----------|---------------------|
| 注意力 | 滑动窗口（4096） | 滑动窗口 |
| 架构 | Dense | MoE（8 专家，激活 2） |
| 参数 | 7B | 47B 总，13B 激活 |

**关键设计：**
- **滑动窗口注意力（SWA）**：每个 token 只关注窗口内的 token——O(n) 复杂度而非 O(n²)
- **MoE（Mixture of Experts）**：多个专家网络，每次只激活 2 个——大容量但低推理成本

### 2.5 Phi-3（Microsoft, 2024）

Phi-3 Mini (3.8B) 在极小的参数量下实现了接近 7B 模型的质量——秘诀是**高质量训练数据**（教科书级别的数据）。

**关键设计：** 数据质量 > 数据量 > 参数量

### 2.6 DeepSeek-V3（DeepSeek, 2024）

| 特性 | DeepSeek-V3 |
|------|------------|
| 参数量 | 671B 总，37B 激活 |
| 架构 | MoE（256 专家，激活 8） |
| 训练成本 | 约 550 万美元（极低成本） |

**关键设计：**
- **MoE + 多头潜在注意力（MLA）**：超大参数量但推理成本低
- **FP8 训练**：极低的训练成本

### 2.7 架构选择的权衡

| 设计决策 | 选项 A | 选项 B | 影响 |
|---------|--------|--------|------|
| 注意力类型 | MHA（标准） | GQA（分组） | GQA 推理快 2-4x |
| 激活函数 | ReLU | SwiGLU | SwiGLU 质量更好 |
| 归一化 | Post-LN | Pre-LN | Pre-LN 训练更稳定 |
| 位置编码 | 绝对位置 | RoPE | RoPE 可扩展到任意长度 |
| 密集 vs MoE | Dense | MoE | MoE 大容量低推理成本 |

## 3. 分析方法

### 如何读一个模型卡

```python
# 1. 加载模型配置
from transformers import AutoConfig
config = AutoConfig.from_pretrained("meta-llama/Llama-3.1-8B-Instruct")

# 2. 打印关键参数
print(f"层数: {config.num_hidden_layers}")
print(f"隐藏维度: {config.hidden_size}")
print(f"注意力头: {config.num_attention_heads}")
print(f"KV头: {config.num_key_value_heads}")
print(f"GQA比例: {config.num_attention_heads // config.num_key_value_heads}")
print(f"词表大小: {config.vocab_size}")
```

## 4. LLM 视角

### 4.1 为什么理解架构很重要

- **选型**：不同任务需要不同架构——代码任务需要大上下文，中文需要大词表
- **微调**：LoRA 需要知道哪些层注入——理解架构才能正确配置
- **部署**：GQA 决定 KV 缓存大小，MoE 决定激活参数量——影响推理成本

### 4.2 2026 年的趋势

- **MoE 成为主流**：DeepSeek-V3 证明 MoE 可以用极低成本训练超大模型
- **长上下文**：128K+ 成为标配——RoPE + YaRN 扩展
- **数据为王**：Phi-3 证明小模型+好数据 > 大模型+差数据

## 5. 工程最佳实践

### 5.1 模型选型指南

| 场景 | 推荐模型 | 原因 |
|------|---------|------|
| 通用英文 | Llama 3.1 8B/70B | 综合质量最好 |
| 中文场景 | Qwen2.5 7B/14B | 中文训练数据最多 |
| 边缘设备 | Phi-3 Mini 3.8B | 极小但质量高 |
| 高吞吐推理 | Mistral/Mixtral | MoE 推理快 |
| 极致质量 | Llama 3.1 405B | 最大开源模型 |

### 5.2 从架构读出部署需求

```
GQA 比例 4:1 → KV 缓存是 MHA 的 1/4 → 更多并发用户
MoE 8 专家激活 2 → 推理只需 13B 参数的计算量 → 推理快
128K 词表 → 嵌入层 128K×4096 ≈ 524M 参数 → 比 32K 多 400M
```

## 6. 面试考点

### Q1：为什么 2026 年的开源 LLM 都用 GQA 而不是 MHA？（难度：⭐⭐）

**参考答案：**
MHA（Multi-Head Attention）中每个查询头都有独立的 K 和 V 头——KV 缓存大小 = 查询头数 × 层 × 序列长度 × 头维度。GQA（Grouped Query Attention）让多个查询头共享一组 KV 头——KV 缓存减少到 1/4（如 Llama 3 的 32:8=4:1）。推理时 KV 缓存是显存瓶颈——GQA 直接减少显存占用、增加并发用户数、加速推理。质量几乎不受影响。

### Q2：MoE 模型的优势和劣势是什么？（难度：⭐⭐⭐）

**参考答案：**
**优势：** (1) 大容量——DeepSeek-V3 有 671B 参数但推理时只激活 37B；(2) 推理快——激活参数量小，计算量与 Dense 13B 相当；(3) 训练效率——更多参数但更少计算。**劣势：** (1) 显存仍然需要存储所有参数——671B 需要 1.3TB；(2) 专家负载不均——某些专家可能很少被激活；(3) 训练不稳定——需要辅助损失确保专家均衡。

### Q3：如果要为中文客服场景选模型，应该选 Llama 3 还是 Qwen2.5？（难度：⭐⭐⭐）

**参考答案：**
选 Qwen2.5。原因：(1) Qwen2.5 在 18T token 上训练，中英文混合——中文 token 效率远高于 Llama 3 的 128K 词表；(2) Qwen2.5 的中文对齐数据更多——SFT/RLHF 在中文上有更多高质量数据；(3) Qwen2.5 的 GQA 统一为 8 KV 头——推理效率稳定。但如果是纯英文场景，Llama 3.1 的英文质量略高。

## 🔑 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|----------|---------|
| GQA | "共享 KV 的注意力" | 分组查询注意力——多个查询头共享一组 KV 头，减少 KV 缓存 |
| SwiGLU | "更好的激活" | Swish + GLU 门控机制——比 ReLU/GELU 质量更好 |
| RMSNorm | "无偏置的归一化" | 没有可学习偏置的 LayerNorm——更快且效果相当 |
| RoPE | "旋转位置编码" | 旋转位置嵌入——支持任意长度外推 |
| MoE | "混合专家" | 多个专家网络，每次只激活部分——大容量低推理成本 |
| 滑动窗口注意力 | "局部注意力" | 每个 token 只关注窗口内的 token——O(n) 复杂度 |

---

## 📚 小结

2026 年开源 LLM 的统一架构：Pre-LN + GQA + SwiGLU + RoPE + RMSNorm。每个选择都有明确的效率或质量理由。MoE 是大模型的新范式——DeepSeek-V3 用 671B 参数但只激活 37B，实现了极低推理成本。数据质量 > 数据量 > 参数量——Phi-3 证明了这一点。理解架构选择是正确选型和部署的基础。
