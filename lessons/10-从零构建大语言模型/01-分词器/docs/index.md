# 分词器——从字符到词元

> LLM 不读文字，只读数字。分词器是文字→数字的桥梁——它决定了模型"看到"什么，以及它错过了什么。

**类型：** 实现课 | **语言：** Python | **前置知识：** 阶段 05 · 04（GloVe/FastText）、阶段 10 · 04（预训练）| **时间：** ~45 分钟

## 1. 问题

神经网络处理数字，不处理字符串。**分词器**将文本转换为整数序列——每个整数对应词表中的一个 token。词表的选择直接决定了：

- 模型能"理解"哪些词（未见过的词 = OOV）
- 序列长度（子词越多→序列越长→越慢）
- 多语言能力（中文每字 1 个 token vs 英文每词 1 个 token）

---

## 2. 概念

### 2.1 BPE 训练过程

```
步骤 1: 将所有单词拆成字符 + </w>
  "lower" → ["l", "o", "w", "e", "r", "</w>"]

步骤 2: 统计所有相邻字符对的频率
  ("l","o"): 5次, ("o","w"): 4次, ("w","e"): 3次

步骤 3: 合并最频繁的对 → "lo"
  "lower" → ["lo", "w", "e", "r", "</w>"]

步骤 4-1000: 重复合并，直到词表大小达到目标（如 32000）
```

### 2.2 特殊 Token

| Token | 用途 | 示例 |
|---|---|---|
| `<BOS>` | 序列开始 | "开始生成了" |
| `<EOS>` | 序列结束 | "生成结束了" |
| `<PAD>` | 填充到固定长度 | 对齐 batch 中的短序列 |
| `[MASK]` | BERT 掩码预测 | "The [MASK] sat on the mat" → 预测 "cat" |
| `<UNK>` | 未登录词（尽量避免） | 词表中没有的词 |

---

## 🔑 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|---|---|---|
| 分词器 | Tokenizer | 将文本转换为整数序列的组件 |
| 子词 | Subword | 介于字符和单词之间的最小语义单元 |
| 词表 | Vocabulary | 分词器能处理的所有 token 的集合 |
| BPE | 字节对编码 | 从字符开始，迭代合并最频繁的相邻对 |

---

## 📚 小结

分词器 = 文字→数字的桥梁。BPE 是最常用的算法——从字符开始迭代合并。词表大小决定了模型能"认识"多少词。特殊 token（BOS/EOS/PAD/MASK）是 LLM 流水线的关键接口。
