# 数据管道——从原始文本到训练批次

> 分词器决定了模型"看到"什么，数据管道决定了"按什么顺序"看。清洗、分词、打包成固定长度的批次——这是预训练的基础设施。

**类型：** 实现课 | **语言：** Python | **前置知识：** 阶段 10 · 01-02（分词器）| **时间：** ~60 分钟

---

## 🎯 学习目标

- [ ] 实现一个完整的 LLM 数据管道——清洗→分词→打包成训练批次
- [ ] 解释 GPT 风格的因果语言建模数据格式——输入是目标序列，标签是右移一位的序列
- [ ] 说明数据质量对预训练的影响——"garbage in, garbage out"在 LLM 中的体现

---

## 1. 问题

有了分词器和原始文本——下一步是把它们转换为模型可以消费的格式。LLM 训练数据需要：

1. **清洗**：去重、过滤低质量文本、移除个人信息
2. **分词**：将文本转换为 token ID 序列
3. **打包**：将变长序列填充/截断为固定长度的批次

**关键洞察：** 数据质量比数据量更重要。Chinchilla 的缩放定律说参数和数据要平衡增长——但前提是数据是高质量的。Llama 3 用 18T token 训练是因为它过滤掉了低质量数据。

---

## 2. 概念

### 2.1 GPT 风格的训练数据格式

```
输入: [token_0, token_1, token_2, ..., token_n-1]
标签: [token_1, token_2, token_3, ..., token_n]
```

**标签是输入右移一位——每个位置预测下一个 token。**

### 2.2 数据质量过滤

| 过滤步骤 | 作用 |
|---|---|
| 去重 | 移除重复文档——防止模型过拟合到重复模式 |
| 语言过滤 | 保留目标语言——移除垃圾/低质量文本 |
| 长度过滤 | 移除太短（<50 token）或太长的文档 |
| 质量启发式 | 基于标点、词频、格式等启发式评分 |

---

## 🔑 关键术语

| 术语 | 含义 |
|---|---|
| 因果语言建模 | 预测下一个 token 的训练目标——输入是标签右移一位 |
| 打包 | 将多个文档拼接成固定长度的训练序列 |
| 去重 | 移除训练数据中的重复文档 |
| 数据配比 | 不同来源数据的混合比例 |

---

## 📚 小结

数据管道 = 清洗→分词→打包。LLM 训练数据格式：输入=标签右移一位。数据质量过滤（去重、语言、长度）比增加数据量更重要。Chinchilla 说参数和数据要平衡增长——但前提是数据质量足够好。
