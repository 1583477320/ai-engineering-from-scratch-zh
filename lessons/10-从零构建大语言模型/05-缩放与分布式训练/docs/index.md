# 缩放与分布式训练

> 从单 GPU 到多 GPU 到多节点——分布式训练是 LLM 规模化的核心工程挑战。

**类型：** 概念课 | **语言：** Python | **前置知识：** 阶段 10 · 04（预训练 MiniGPT）| **时间：** ~60 分钟

---

## 🎯 学习目标

- [ ] 解释数据并行 vs 模型并行 vs 张量并行的区别
- [ ] 说明 DeepSpeed ZeRO 如何将 3B 模型的显存需求从 24GB 降到 4GB
- [ ] 理解梯度检查点——用计算换内存的权衡

---

## 1. 问题

训练 124M 的 MiniGPT 在单 GPU 上几小时就完成。但 Llama 3 8B 需要 ~8×80GB 的 GPU 才能训练——因为需要存储：模型参数（32GB）+ 优化器状态（64GB）+ 激活值（~20GB）= >100GB。分布式训练将这些负载分散到多个 GPU/节点上。

---

## 2. 概念

### 2.1 三种并行方式

| 并行方式 | 切分什么 | 通信开销 | 适用 |
|---|---|---|---|
| **数据并行** | 每个 GPU 有完整模型副本 | 梯度同步 | 模型放得下单 GPU |
| **张量并行** | 每层的矩阵按行切分 | 每层内通信 | 模型太大放不下但单层放得下 |
| **流水线并行** | 模型按层切分 | 层间通信 | 模型太大需要跨 GPU |

### 2.2 DeepSpeed ZeRO

| 阶段 | 切分什么 | 显存节省 |
|---|---|---|
| ZeRO-1 | 优化器状态 | 4× |
| ZeRO-2 | 优化器 + 梯度 | 8× |
| ZeRO-3 | 优化器 + 梯度 + 参数 | N×（N=GPU数） |

### 2.3 梯度检查点

**问题：** 激活值占用大量显存（~20GB for 3B model）
**解决：** 只保存部分层的激活值，其余在反向传播时重新计算。用 ~30% 的额外计算时间换取 ~70% 的显存节省。

---

## 🔑 关键术语

| 术语 | 含义 |
|---|---|
| 数据并行 | 每个 GPU 有完整模型，处理数据的不同部分 |
| 张量并行 | 每层的权重矩阵按行切分到多个 GPU |
| ZeRO | 深度学习优化器——将优化器/梯度/参数切分到多个 GPU |
| 梯度检查点 | 用计算时间换显存——不保存中间激活值，反向传播时重新计算 |

---

## 📚 小结

分布式训练 = 数据并行 + 张量并行 + 流水线并行。ZeRO 将显存需求从单 GPU 扩展到多 GPU。梯度检查点用计算时间换显存。2026 年 3B 模型在单 GPU 上可训练，8B 需要 2-4 GPU，70B 需要 32-64 GPU。

---

## ✏️ 练习

1. 在 2 个 GPU 上实现数据并行——对比单 GPU 的训练速度和收敛
2. 用 DeepSpeed ZeRO-3 在 2 个 GPU 上训练 3B 模型——测量显存使用
