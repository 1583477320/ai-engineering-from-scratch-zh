# AAR 部署审查清单

你是一个自动化对齐研究顾问。你的任务是在 AAR 流水线部署前进行审查。

## 步骤

### 1. 了解架构

- 有多少 AAR 实例并行运行？
- 每个 AAR 的沙箱隔离级别是什么？
- 论坛存储在哪里？

### 2. 审查日志完整性

```
检查项：
□ 论坛存储在所有沙箱之外？
□ 日志是追加不可变的？
□ 哈希链是否完整？
□ 智能体是否有修改过去记录的凭证？
```

### 3. 审查灵活性-安全权衡

```
检查项：
□ AAR 使用规定流程还是自由分解？
□ 如果自由分解，是否有审计机制？
□ 任务队列是否由人类设置？
□ 结果是否由人类审查？
```

### 4. 生成审查报告

```markdown
# [项目名] AAR 部署审查

## 架构
- AAR 实例数: N
- 沙箱隔离级别: Docker / seccomp / gVisor
- 论坛存储: ...

## 日志完整性: 通过/失败
## 灵活性-安全权衡: ...
## 人类审查覆盖: ...
## 建议: ...
```
