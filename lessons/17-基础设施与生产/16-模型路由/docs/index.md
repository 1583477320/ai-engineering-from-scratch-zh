# 模型路由作为成本降低原语

> 动态代理评估每个请求（任务类型、词元长度、嵌入相似度、置信度），将简单查询发送到廉价模型，将复杂查询升级到前沿模型。也称为模型级联。生产案例显示在相同质量下跨部署环境节省 20-60%；高流量 SaaS 上 30% 的路由效率提升转化为六位数的年节省。2026 年的背景是 LLM 推理价格每年下降约 10 倍——GPT-4 级词元从 $20/M 降到约 $0.40/M。路由是你将价格下降转化为利润的方式。

**类型：** 概念课
**语言：** Python
**前置知识：** 阶段 17 · 01（托管 LLM 平台）、阶段 17 · 19（AI 网关）
**预计时间：** ~60 分钟
**所处阶段：** Tier 3

## 1. 问题

你的服务每月在 GPT-5 上花费 $80K。分析显示 70% 的查询是简单的："巴黎现在几点？""重新表述这个句子。"Haiku 级模型以 3% 的成本完美处理这些。30% 需要 GPT-5 的推理能力——编码、数学、多步规划。

如果将 70% 路由到廉价、30% 路由到昂贵，账单在相同产品品质下降低约 65%。这就是路由。技巧是构建代理而不降低质量。

## 2. 概念

### 2.1 四个路由信号

1. **任务分类：** 简单/复杂/代码生成/数学/聊天。可以是基于规则的分类器、小型 LLM（Haiku 级，$0.25/M）、或嵌入相似度到标注桶。输出：路由 = 廉价 / 平衡 / 前沿。

2. **提示词长度：** 超过 4K 词元的提示词通常需要前沿模型保持连贯性。少于 500 词元的通常不需要。

3. **已知困难集合的嵌入相似度：** 如果查询与已知困难桶的余弦相似度 > 0.88，直接升级到前沿。

4. **首次通过的自置信度：** 发送到廉价模型；如果模型的 log-probs 显示低置信度或它拒绝或输出模糊语言，在前沿模型上重试。增加约 10% 流量的 P95 延迟，但节省其余 90% 的 50%+。

### 2.2 三种模式

**预路由（分类器在前）：** 增加约 5-10ms 延迟；总体最快。

**级联（廉价优先，低置信度时升级）：** 中位延迟约 1.2 倍（廉价运行+验证），升级请求约 2 倍。质量下限最高。

**集成路由（廉价和前沿并行运行，奖励模型选择）：** 质量最高，成本最高；仅用于关键 A/B 测试。

### 2.3 2026 年价格曲线

| 模型级别 | 2022 年底 | 2026 年 | 变化 |
|---|---|---|---|
| GPT-4 级质量 | ~$20/M | ~$0.40/M | 便宜 50 倍 |
| 前沿（GPT-5、Claude 4） | — | ~$3-10/M | 新层级 |

大部分改进来自服务效率——第 17 · 04-09 课的核心经验转化为了提供商端的成本下降。路由让你在应用层捕获这些收益，而不是等待所有用户迁移到廉价层级。

### 2.4 漂移是真正的风险

你的路由将 40% 发送到廉价模型。六个月后，任务分布迁移（用户变得更复杂，问题更长）。路由器没有注意到，因为它的分类器是基于 Q1 数据训练的。质量悄然下降。没有人抱怨得够大声。你在竞争对手的基准测试中发现自己输了。

按在线质量指标门控路由：
- 用户点赞/踩按路由分
- 在留存样本（5%）上自动 LLM 裁判按路由分
- 升级率：如果级联向上路由 > 30%，廉价模型被过度路由
- 每路由的拒绝率

## 3. 从零实现

### 第 1 步：简单任务分类器

```python
def classify_task(prompt):
    """基于长度和关键词的简单任务分类。"""
    keywords_complex = {"解释", "分析", "为什么", "如何", "compare", "analyze"}
    keywords_code = {"代码", "代码", "debug", "function", "class"}

    prompt_lower = prompt.lower()

    for kw in keywords_code:
        if kw in prompt_lower:
            return "frontier"
    for kw in keywords_complex:
        if kw in prompt_lower:
            return "balanced"
    if len(prompt) > 4000:
        return "balanced"

    return "cheap"
```

完整代码见 `code/main.py`。

## 5. 工程最佳实践

### 5.1 先离线评估，再在线部署

用标注数据集评估路由策略——对比廉价模型、前沿模型和路由后混合的质量。确认质量损失在容忍范围内再上线。

### 5.2 在线监控是必须的

离线评估不能捕获分布漂移。上线后用在线指标（点赞/踩、LLM 裁判、升级率）持续监控。

### 5.3 中文场景特别建议

- **中文任务分类更难。** 英文的"simple/complex"关键词在中文中表达更多样。建议用小型中文 LLM 做分类
- **中文模型的价格差异。** 国内中文 LLM 的价格差异比英文更大——深度求索、通义千问等开源模型免费，而 GPT-4o 的中文价格较高。路由的收益可能更大
- **中文嵌入模型。** 已知困难集合的嵌入相似度需要用中文嵌入模型。text-embedding-v3（阿里云）、bge-large-zh（智源）是常用选择

## 6. 常见错误

### 错误 1：离线评估通过但在线质量下降

**现象：** 离线评估显示路由策略质量损失 < 1%。上线两个月后用户投诉质量下降。

**原因：** 分布漂移——任务分布从简单转向复杂，路由分类器基于旧数据训练。

**修复：** 上线后持续用在线指标监控——升级率、LLM 裁判分数、用户反馈。

### 错误 2：所有查询都路由到廉价模型

**现象：** 成本降了 60%，但 30% 的复杂查询质量下降 3-5 点。

**原因：** 路由分类器过于激进——将中等复杂度的查询也路由到廉价模型。

**修复：** 调整分类阈值——宁可多路由到前沿（成本增加 10%），也不要降低质量。

## 7. 面试考点

### Q1：模型级联和预路由的核心区别是什么？（难度：⭐⭐）

**参考答案：**
预路由：分类器在前，每个请求被分类后直接发送到对应模型——不会重新运行。延迟增加约 5-10ms。级联：廉价优先运行，检查置信度；低置信度时升级到前沿——需要运行两次。中位延迟约 1.2 倍。预路由更快但质量下限低（分类器可能误判）；级联更慢但质量下限高（廉价模型的输出如果不够好可以升级）。

### Q2：模型路由的最大风险是什么？如何缓解？（难度：⭐⭐⭐）

**参考答案：**
最大风险是漂移——廉价模型在上线后质量悄然下降。六个月后任务分布迁移（用户更复杂），路由分类器基于旧数据训练，没有注意到变化。缓解：在线质量门控——每路由持续监控升级率（>30% 说明廉价模型被过度路由）、LLM 裁判在留存样本上的分数、用户反馈。不要只依赖离线评估。

## 🔑 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|---|---|---|
| 模型路由 | "成本代理" | 每请求动态选择模型 |
| 模型级联 | "廉价优先升级" | 先运行廉价模型，低置信度时升级到前沿 |

| 漂移 | "廉价爬升" | 分布迁移而路由器未注意 |
| 在线质量门控 | "实时检查" | 自动 LLM 裁判抽样生产流量 |

## 📚 小结

模型路由通过动态选择模型来降低成本——简单查询发送到廉价模型，复杂查询升级到前沿。三种模式：预路由（最快）、级联（质量最好）、集成路由（最贵）。关键风险是漂移——上线后廉价模型的质量悄然下降。缓解：在线质量门控，不只是离线评估。路由是将 LLM 价格下降（每年约 10 倍）转化为利润的最直接方式。
