根据工作负载特征分流到交互/半交互/批处理，并估算节省。

**输出：**

1. **工作负载分类。** 将每个工作负载分为交互式/半交互式/批处理。
2. **SLA 分析。** 每个工作负载的 SLA 是什么？24 小时可接受吗？
3. **成本估算。** 对比批处理+缓存 vs 同步无缓存的成本。
4. **提供商选择。** OpenAI/Anthropic/Google 的批处理 API 对比。
5. **实施计划。** 分阶段迁移路径。

**硬拒绝：**
- 将所有生产工作负载分类为交互式——SLA 才是标准，不是"生产环境"标签。
- 不叠加缓存就使用批处理——批处理+缓存的叠加效果远大于单独使用。

**输出格式：** 工作负载分流报告——分类、SLA 分析、成本估算、提供商选择、实施计划。
