# 位置编码

> 自注意力本身不知道词序。位置编码告诉每个词元"我在序列的哪个位置"——没有它，"猫坐在垫子上"和"垫子坐在猫上"是同一个向量。

## 1. 问题

"猫坐在垫子上"和"垫子坐在猫上"——用纯自注意力处理这两个句子，它们得到完全相同的表示。因为自注意力只计算词元之间的相似度，不考虑词元在序列中的位置。

**位置编码解决这个问题。** 将位置信息注入每个词元的嵌入——让模型知道"我在序列的第 3 个位置"。

---

## 2. 概念

### 2.1 正弦位置编码

原始 Transformer 使用正弦函数：

$$PE_{(pos, 2i)} = \sin\left(\frac{pos}{10000^{2i/d_{model}}}\right)$$
$$PE_{(pos, 2i+1)} = \cos\left(\frac{pos}{10000^{2i/d_{model}}}\right)$$

**为什么用 sin/cos？** 两个性质：(1) 每个维度都是一个不同频率的正弦波——覆盖从低频（远距离）到高频（近距离）的位置信息；(2) 相对位置可以用线性变换表达——$PE_{pos+k}$ 可以表示为 $PE_{pos}$ 的线性函数。

### 2.2 为什么不是可学习的编码

正弦编码有两个优势：(1) 可以外推到比训练时更长的序列；(2) 无额外参数（可学习编码有 $L \times d_{model}$ 个参数）。

RoPE（旋转位置编码，GPT-4/LLaMA 使用）进一步改进：将位置信息编码为旋转矩阵，让相对位置信息在注意力计算中直接体现——无需额外参数。

## 3. 从零实现

```python
import numpy as np
import math

def sinusoidal_position_encoding(max_len, d_model):
    """正弦位置编码——每个位置一个 d_model 维向量。"""
    pe = np.zeros((max_len, d_model))
    for pos in range(max_len):
        for i in range(0, d_model, 2):
            angle = pos / (10000 ** (i / d_model))
            pe[pos, i] = math.sin(angle)
            pe[pos, i + 1] = math.cos(angle)
    return pe

# 可视化：位置 0 和位置 10 在每个维度上的编码值
pe = sinusoidal_position_encoding(100, 64)
print("位置 0 前 8 维:", pe[0, :8].round(3))
print("位置 10 前 8 维:", pe[10, :8].round(3))
print("位置 0 和位置 10 的差 = 位置 10 的相对编码（线性变换）")
```

## 4. 常见错误

### 错误 1：正弦编码和可学习编码混用

**现象：** 模型在长序列上突然崩溃。

**原因：** 正弦编码可以外推到更长序列；可学习编码固定在训练长度。如果训练长度 512，推理长度 1024——可学习编码完全失效。

### 错误 2：忘记加位置编码到输入嵌入

**现象：** 两个不同顺序的句子产生完全相同的表示。

**原因：** 自注意力对输入是置换不变的——它只知道词元之间的相似度，不知道谁在前面。位置编码是位置信息的唯一来源。

## 🔑 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|---|---|---|
| 正弦位置编码 | "用 sin/cos 编码位置" | 不同频率的正弦波组合，覆盖远近位置信息 |
| RoPE | "旋转位置编码" | 将位置信息编码为旋转矩阵，相对位置在注意力中直接体现 |
| 可学习编码 | "把位置当参数学" | 训练时学习的位置嵌入。无外推能力 |

---

## 📚 小结

自注意力对输入顺序是置换不变的——需要位置编码注入位置信息。正弦编码用不同频率的 sin/cos 覆盖位置空间，可外推。RoPE 将位置编码为旋转矩阵，相对位置信息更直接。没有位置编码，"猫坐在垫子上"和"垫子坐在猫上"无法区分。
