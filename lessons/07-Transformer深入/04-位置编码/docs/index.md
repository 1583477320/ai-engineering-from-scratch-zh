## 1. 问题

"猫坐在垫子上"和"垫子坐在猫上"——用纯自注意力处理这两个句子，它们得到完全相同的表示。因为自注意力只计算词元之间的相似度，不考虑词元在序列中的位置。

**位置编码解决这个问题。** 将位置信息注入每个词元的嵌入——让模型知道"我在序列的第 3 个位置"。

---

## 2. 概念

### 2.1 正弦位置编码

原始 Transformer 使用正弦函数：

$$PE_{(pos, 2i)} = \sin\left(\frac{pos}{10000^{2i/d_{model}}}\right)$$
$$PE_{(pos, 2i+1)} = \cos\left(\frac{pos}{10000^{2i/d_{model}}}\right)$$

**为什么用 sin/cos？** 两个性质：(1) 每个维度都是一个不同频率的正弦波——覆盖从低频（远距离）到高频（近距离）的位置信息；(2) 相对位置可以用线性变换表达——$PE_{pos+k}$ 可以表示为 $PE_{pos}$ 的线性函数。

### 2.2 为什么不是可学习的编码

正弦编码有两个优势：(1) 可以外推到比训练时更长的序列；(2) 无额外参数（可学习编码有 $L \times d_{model}$ 个参数）。

RoPE（旋转位置编码，GPT-4/LLaMA 使用）进一步改进：将位置信息编码为旋转矩阵，让相对位置信息在注意力计算中直接体现——无需额外参数。

## 3. 从零实现

```python
import numpy as np
import math

def sinusoidal_position_encoding(max_len, d_model):
    """正弦位置编码——每个位置一个 d_model 维向量。"""
    pe = np.zeros((max_len, d_model))
    for pos in range(max_len):
        for i in range(0, d_model, 2):
            angle = pos / (10000 ** (i / d_model))
            pe[pos, i] = math.sin(angle)
            pe[pos, i + 1] = math.cos(angle)
    return pe

# 可视化：位置 0 和位置 10 在每个维度上的编码值
pe = sinusoidal_position_encoding(100, 64)
print("位置 0 前 8 维:", pe[0, :8].round(3))
print("位置 10 前 8 维:", pe[10, :8].round(3))
print("位置 0 和位置 10 的差 = 位置 10 的相对编码（线性变换）")
```

完整代码见 `code/main.py`——纯 NumPy，可立即运行。

## 4. 工业工具

### 4.1 PyTorch 位置编码

```python
import torch.nn as nn

# 正弦位置编码
d_model = 512
max_len = 512
pe = nn.Parameter(torch.randn(max_len, d_model) * 0.02)

# 可学习位置编码
position_ids = torch.arange(max_len).unsqueeze(0)  # (1, max_len)
position_embeddings = pe(position_ids)  # (1, max_len, d_model)
```

### 4.2 HuggingFace Transformers

```python
from transformers import AutoModel

# 加载预训练模型（已包含位置编码）
model = AutoModel.from_pretrained("bert-base-chinese")

# 查看位置编码
print(f"位置编码形状: {model.embeddings.position_embeddings.weight.shape}")
# 位置编码形状: torch.Size([512, 768])
```

### 4.3 性能对比

| 位置编码 | 参数量 | 外推能力 | 相对位置 | 适用模型 |
|---|---|---|---|---|
| 正弦编码 | 0 | ✓ | 间接 | 原始 Transformer |
| 可学习编码 | L × d_model | ✗ | 间接 | BERT, GPT-2 |
| RoPE | 0 | ✓ | 直接 | GPT-4, Llama 3 |

## 5. LLM 视角

### 5.1 在主流大语言模型中的体现

GPT-4、Claude、Llama 3 等大语言模型都使用 RoPE（旋转位置编码）来编码位置信息。RoPE 的核心优势是：将位置信息编码为旋转矩阵，让注意力分数直接反映相对位置——无需额外参数。

你在本课实现的正弦编码是 RoPE 的理论基础。理解正弦编码的频率分解思想，就能理解 RoPE 为什么有效。

### 5.2 LLM 时代什么变了？

**位置编码从"可选"变成"必需"。** 原始 Transformer 的正弦编码是可选的（理论上可以不用）。但现代大语言模型必须使用位置编码——因为自回归生成依赖位置信息。

**外推能力变得至关重要。** GPT-4 支持 128K 上下文窗口。如果位置编码不能外推，模型在推理时遇到更长的序列就会崩溃。RoPE 的外推能力让这成为可能。

**相对位置信息更直接。** 正弦编码的相对位置是间接的（需要线性变换）。RoPE 的相对位置是直接的（旋转矩阵的性质）。这让模型更容易学习"距离 3 个位置的关系"这样的模式。

### 5.3 什么没变？

**核心思想没变。** 从正弦编码到 RoPE，核心思想都是"用不同频率的信号编码位置信息"。正弦编码用 sin/cos 函数，RoPE 用旋转矩阵——数学形式不同，但思想一致。

**位置信息必须注入。** 没有哪个大语言模型跳过了位置编码。自注意力的置换不变性是根本问题，位置编码是唯一解决方案。

### 5.4 使用 ChatGPT / Claude 时的直接体验

当你在 ChatGPT 中输入一段话，模型生成回复时，它在每个生成步骤都在使用位置编码——每个新生成的词元都要知道"我在序列的哪个位置"。

如果你输入 100 个词元，模型生成 50 个词元，那么位置编码总共处理了 150 个位置。如果没有位置编码，模型就无法区分"第一个词元"和"第 150 个词元"——生成结果会完全混乱。

## 6. 工程最佳实践

### 6.1 工业界常用方案

| 场景 | 推荐方案 | 备注 |
|---|---|---|
| 学习 / 实验 | 正弦编码 | 简单易理解 |
| 固定长度任务 | 可学习编码 | 可能学到更好的位置表示 |
| 长序列 / 生成任务 | RoPE | 外推能力强，相对位置信息更直接 |
| 生产环境 | RoPE + 缩放 | 支持 128K+ 上下文窗口 |

### 6.2 中文场景特别建议

- 中文分词器训练时，确保训练语料中中文占比 ≥ 30%，否则中文会被过度切分
- 使用 `bert-base-chinese` 的分词器时注意：它是字级别的，不产生子词
- 中文 + 代码混合场景，优先使用 Llama 3 或 GPT-4 的分词器（128K+ 词表，对多语言更友好）

### 6.3 踩坑经验

- 训练分词器时未设置 `byte_fallback=True`，导致部署时遇到生僻字直接报错
- `padding_side="left"` 还是 `"right"`？自回归模型用 left，编码器模型用 right
- 不要用训练集的平均长度作为 `max_length`——留 20% 余量，否则截断率太高
- 位置编码的维度必须与嵌入维度一致，否则无法相加
- 外推时注意位置编码的最大长度——超出范围的位置编码会失效

## 7. 常见错误

### 错误 1：正弦编码和可学习编码混用

**现象：** 模型在长序列上突然崩溃。

**原因：** 正弦编码可以外推到更长序列；可学习编码固定在训练长度。如果训练长度 512，推理长度 1024——可学习编码完全失效。

**修复：**
```python
# ❌ 错误写法：混用两种编码
x = embedding + sinusoidal_pe + learned_pe  # 位置信息混乱

# ✓ 正确写法：只使用一种
x = embedding + sinusoidal_pe  # 或
x = embedding + learned_pe
```

### 错误 2：忘记加位置编码到输入嵌入

**现象：** 两个不同顺序的句子产生完全相同的表示。

**原因：** 自注意力对输入是置换不变的——它只知道词元之间的相似度，不知道谁在前面。位置编码是位置信息的唯一来源。

**修复：**
```python
# ❌ 错误写法：没有位置编码
x = embedding[input_ids]  # 位置信息丢失

# ✓ 正确写法：加上位置编码
x = embedding[input_ids] + positional_encoding[:seq_len]
```

### 错误 3：位置编码维度不匹配

**现象：** 运行时报错 `ValueError: operands could not be broadcast together`。

**原因：** 位置编码的维度必须与嵌入维度一致。如果 d_model=512，位置编码也必须是 512 维。

**修复：**
```python
# ❌ 错误写法：维度不匹配
embedding = np.random.randn(100, 768)  # 768 维
pos_encoding = np.random.randn(100, 512)  # 512 维
x = embedding + pos_encoding  # 报错

# ✓ 正确写法：维度一致
embedding = np.random.randn(100, 512)  # 512 维
pos_encoding = np.random.randn(100, 512)  # 512 维
x = embedding + pos_encoding  # 正常
```

### 错误 4：外推时超出最大长度

**现象：** 推理时遇到更长的序列，模型输出异常。

**原因：** 正弦编码可以外推，但可学习编码固定在训练长度。如果训练长度 512，推理长度 1024——可学习编码完全失效。

**修复：**
```python
# ❌ 错误写法：超出最大长度
max_len = 512
pe = LearnedPositionEncoding(max_len, d_model)
x = pe.forward(1024)  # 报错或输出异常

# ✓ 正确写法：使用可以外推的编码
pe = sinusoidal_position_encoding(1024, d_model)  # 正弦编码可以外推
x = pe[:1024]
```

## 8. 面试考点

### Q1：为什么自注意力需要位置编码？（难度：⭐⭐）

**参考答案：**
自注意力对输入顺序是置换不变的——它只计算词元之间的相似度，不考虑词元在序列中的位置。"猫坐在垫子上"和"垫子坐在猫上"用纯自注意力处理会得到完全相同的表示。位置编码将位置信息注入每个词元的嵌入，让模型知道"我在序列的第 3 个位置"。

### Q2：正弦编码为什么可以用不同频率的 sin/cos？（难度：⭐⭐）

**参考答案：**
正弦编码用不同频率的 sin/cos 函数编码位置信息。低频（长波长）编码远距离位置关系，高频（短波长）编码近距离位置关系。这就像用不同精度的尺子测量距离——高频维度测量"相邻位置"，低频维度测量"远距离位置"。

### Q3：RoPE 如何编码相对位置？（难度：⭐⭐⭐）

**参考答案：**
RoPE 将位置信息编码为旋转矩阵。对于位置 m 和位置 n，它们的相对位置信息通过旋转矩阵的性质直接体现在注意力分数中：$q_m \cdot k_n = f(x_m, x_n, m-n)$。这意味着注意力分数只依赖相对位置 $m-n$，而不依赖绝对位置 $m$ 和 $n$。

### Q4：可学习编码和正弦编码的优缺点是什么？（难度：⭐⭐）

**参考答案：**
可学习编码的优点是可能学到更好的位置表示；缺点是固定在训练长度，无法外推，有 $L \times d_{model}$ 个参数。正弦编码的优点是无额外参数，可以外推到更长序列；缺点是相对位置信息是间接的（需要线性变换）。

### Q5：位置编码的维度必须与嵌入维度一致吗？（难度：⭐⭐）

**参考答案：**
是的。位置编码必须与嵌入维度一致，因为位置编码要与词元嵌入相加。如果维度不一致，无法进行逐元素相加。这是位置编码的基本约束。

## 🔑 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|---|---|---|
| 正弦位置编码 | "用 sin/cos 编码位置" | 不同频率的正弦波组合，覆盖远近位置信息 |
| RoPE | "旋转位置编码" | 将位置信息编码为旋转矩阵，相对位置在注意力中直接体现 |
| 可学习编码 | "把位置当参数学" | 训练时学习的位置嵌入。无外推能力 |
| 位置嵌入 | "位置向量" | 每个位置一个 d_model 维向量，与词元嵌入相加 |
| 外推能力 | "超出训练长度还能用" | 位置编码能否处理比训练时更长的序列 |
| 相对位置 | "两个词元之间的距离" | 位置 m 和位置 n 的相对距离 m-n |

## 📚 小结

自注意力对输入顺序是置换不变的——需要位置编码注入位置信息。正弦编码用不同频率的 sin/cos 覆盖位置空间，可外推。RoPE 将位置编码为旋转矩阵，相对位置信息更直接。没有位置编码，"猫坐在垫子上"和"垫子坐在猫上"无法区分。
