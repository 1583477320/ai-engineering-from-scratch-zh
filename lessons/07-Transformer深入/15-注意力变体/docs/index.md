# 注意力变体

> 缩放点积注意力是默认，但不是唯一。线性注意力用核方法把 O(N²) 压到 O(N)；滑动窗口让模型在超长序列上也能工作；稀疏注意力只关注最相关的词元。

**类型：** 概念课 | **语言：** Python | **前置知识：** 阶段 07 · 05（完整 Transformer）| **时间：** ~60 分钟

## 1. 问题

标准 Transformer 的注意力是 O(N²)——128K 上下文需要约 128GB 内存（单精度）。这在生产中几乎不可能。各种注意力变体被发明来解决这个问题。

---

## 2. 概念

### 2.1 六种主要注意力变体

| 变体 | 核心思想 | 复杂度 | 使用场景 |
|---|---|---|---|
| **缩放点积** | 标准注意力 | O(N²) | 所有场景 |
| **线性注意力** | 核函数替代 softmax | O(N) | 长序列生成 |
| **滑动窗口** | 每个位置只关注 W 个邻居 | O(NW) | 超长文本 |
| **分组查询注意力** | 多个 Q 共享 K/V | 减少 KV 缓存 | LLaMA, Mistral |
| **多查询注意力** | 所有 Q 共享一对 K/V | 极大减少 KV 缓存 | 边缘推理 |
| **稀疏注意力** | 只计算最重要的 N×M 个分数 | O(NM), M<<N | 长文档 |

### 2.2 GQA——2026 的主流选择

```
标准注意力:  8 个头 × 8 个 KV 对 = 64 个 KV 缓存
GQA:         8 个头 × 2 个 KV 对 = 16 个 KV 缓存
MQA:         8 个头 × 1 个 KV 对 = 8 个 KV 缓存
```

LLaMA 3、Mistral、Qwen2.5 都使用 GQA——在保持质量的前提下将 KV 缓存降低 4 倍。

---

## 🔑 关键术语

| 术语 | 含义 |
|---|---|
| 线性注意力 | 用核函数替代 softmax，将复杂度从 O(N²) 降到 O(N) |
| 滑动窗口注意力 | 每个位置只关注 W 个邻居——超长序列的近似方案 |
| GQA | 分组查询注意力——多组 Q 共享同一组 KV，减少内存 |
| 稀疏注意力 | 只计算最重要的 N×M 个分数——Longformer、BigBird |

---

## 📚 小结

标准注意力 O(N²) 限制了上下文长度。线性注意力 O(N) 但损失精确度。滑动窗口 O(NW) 在窗口内是精确的但丢失长距离信息。GQA 在多头注意力上减少 KV 缓存——2026 年 LLaMA、Mistral、Qwen 的默认选择。没有一种变体完美——实际部署中组合使用（如 GQA + 滑动窗口 + Flash Attention）。
