# 推测解码速查指南

> 快速回顾推测解码的核心机制、配置要点和工程实践。

---

## 核心机制

```
草稿模型（小）                验证模型（大）
    │                            │
    ├─ 生成候选 t1 ──────────────┤
    ├─ 生成候选 t2 ──────────────┤
    ├─ 生成候选 t3 ──────────────┤
    ├─ 生成候选 t4 ──────────────┤
    │                            ├─ 一次前向验证 [t1, t2, t3, t4]
    │                            │
    │                            ├─ t1 匹配 → 接受
    │                            ├─ t2 匹配 → 接受
    │                            ├─ t3 不匹配 → 拒绝，从目标分布重采样
    │                            └─ t3 之后全部丢弃
```

## 关键公式

| 公式 | 含义 |
|---|---|
| 接受概率 = min(1, P_target(t) / P_draft(t)) | 草稿词元被接受的概率 |
| 拒绝后重采样 ∝ max(P_target - P_draft, 0) | 拒绝后从残差分布重采样 |
| 加速比 ≈ K / (1 - α) | α 为拒绝率，K 为草稿长度 |

## 模型配对建议

| 草稿模型 | 验证模型 | 典型加速比 | 适用场景 |
|---|---|---|---|
| 0.5B | 70B | 2-3x | 通用文本生成 |
| 1.5B | 70B | 1.5-2.5x | 代码补全 |
| 同模型浅层 | 同模型完整 | 1.3-1.8x | 无额外模型可用 |
| 专用小模型 | 通用大模型 | 2-4x | 特定领域（如数学） |

## 草稿长度 K 的选择

| K 值 | 优势 | 劣势 |
|---|---|---|
| K=1 | 最安全，无浪费 | 加速有限 |
| K=3-5 | 平衡点，大多数场景最优 | 中等计算浪费 |
| K=8+ | 理论加速更大 | 拒绝率高时浪费严重 |

## 部署工具速查

| 工具 | 命令/配置 | 备注 |
|---|---|---|
| llama.cpp | `./main -m model.gguf --draft draft.gguf --draft-p 5` | 本地部署首选 |
| vLLM | `SpeculativeDecoding(draft_model="...")` | 在线服务 |
| HuggingFace | `model.generate(speculative=True)` | 实验/研究 |
| TensorRT-LLM | 配置 draft model section | NVIDIA GPU 优化 |

## 常见陷阱

1. **草稿模型太差**：拒绝率 > 60%，加速比反而 < 1x（比不用还慢）
2. **草稿模型太大**：草稿生成时间超过验证时间，失去加速意义
3. **KV 缓存不共享**：两个模型各自维护 KV 缓存，显存翻倍
4. **忽略额外前向**：全部接受时需要额外一次目标模型前向生成下一个词元

## 分布等价性验证方法

```
1. 用标准解码生成 N 条序列（如 N=5000）
2. 用推测解码生成 N 条序列
3. 统计每种 3-gram 的频率
4. 计算两组频率的最大差异
5. 如果最大差异 < 0.05（N=5000 时），认为分布等价
```
