# AI文档审计技能

## 目标
给定模型卡、数据表或系统卡，审计章节覆盖、数值分解和可验证证明的存在性。

## 审计流程

### 1. 章节完整性检查

**模型卡必填章节：**
- [ ] 模型详情
- [ ] 预期用途
- [ ] 因素
- [ ] 指标
- [ ] 评估数据
- [ ] 训练数据
- [ ] 量化分析
- [ ] 伦理考量
- [ ] 注意事项和建议

**数据表必填章节：**
- [ ] 动机
- [ ] 组成
- [ ] 收集过程
- [ ] 标注
- [ ] 用途
- [ ] 分发
- [ ] 维护

**系统卡必填章节：**
- [ ] 部署
- [ ] 安全能力
- [ ] 一致性
- [ ] 事件响应
- [ ] 监管一致性

### 2. 数值分解检查

**量化分析验证：**
- 指标是否按敏感属性分解？
- 分解是否涵盖所有相关因素？
- 差异是否在统计上显著？

### 3. 可验证证明检查

**证明类型：**
- 密码签名
- 硬件TEE证明
- 可验证计算证明

**证明覆盖范围：**
- 评估结果
- 数据来源
- 训练过程

### 4. 改进建议

**缺失章节：**
- 识别缺失的必填章节
- 提供填充建议

**弱章节：**
- 识别占位符内容
- 提供具体证据建议

**证明缺失：**
- 识别可验证证明的空白
- 推荐证明实施
