# 可扩展监督与弱到强泛化

> Burns et al.（OpenAI Superalignment, 2023）提出了超对齐问题的代理：用弱模型产生的标签微调强模型。如果强模型从不完美的弱监督中正确泛化，当前人类规模的对齐方法可能扩展到超人系统。可扩展监督和弱到强泛化是互补的。可扩展监督（辩论、递归奖励建模、任务分解）增加监督者的能力以跟上被监督的模型。W2SG 确保强模型从监督者提供的任何不完美信号中正确泛化。

## 1. 问题

到目前为止的每个对齐技术都假设监督者能够评估模型的行为。当模型是超人时，监督者就成了薄弱环节。超对齐问题：较弱的监督者能否可靠地产出一个更强的、对齐的模型？

Burns et al. 将其简化为一个可操作的经验设置：用弱模型监督强模型，测量弱监督保留了多少强模型的能力。这不是超对齐的解决方案——它是一个让我们测量进展的代理。

---

## 2. 概念

### 2.1 W2SG：Burns et al. 的设置

- 弱模型：GPT-2 级
- 强模型：GPT-4 级
- 目标：强 GPT-4 在任务上的上限

流程：
1. 获取弱模型在任务上的零样本预测
2. 在弱标签数据上微调强模型
3. 测量强微调模型的准确率

差距指标：性能差距恢复 PGR = (微调后 - 弱) / (天花板 - 弱)。PGR=1.0 意味着弱监督完全闭合了差距；PGR=0 意味着弱监督没有帮助。

### 2.2 经验发现

在 NLP、棋谜和奖励建模任务上，PGR 持续为正：强模型从不完美的标签中泛化。任务特定 PGR 从约 20% 到约 80% 不等。强模型从不完美的标签中"理解"了预期任务，用预训练先验填充。

局限：W2SG 中的弱-强差距是能力差距，不是对齐差距。弱模型是对齐的；真正超人的模型可能被人类对齐但有更大的能力差距。强模型的泛化可能更多关于任务先验而非恢复真实意图。

### 2.3 可扩展监督：三种机制

- **辩论（Irving et al., 2018）。** 两个 U 的实例辩论对立面；弱裁判决定。假设：找到令人信服的真实答案比找到令人信服的虚假答案更容易
- **递归奖励建模（Leike et al., 2018）。** U 帮助人类为 U+1 训练奖励模型。监督者的能力随模型增长
- **任务分解（Christiano, Shlegeris, Amodei 2018）。** 将困难任务分解为人类可以检查的子任务，递归进行。假设可分解性

每种机制假设了任务结构或中间组件对齐的某种特性。

### 2.4 为什么可扩展监督和 W2SG 是互补的

可扩展监督增加监督者的有效信号质量。W2SG 从监督者提供的任何不完美信号中闭合差距。Lang et al.（arXiv:2501.13124）将两者结合：辩论协议提供更好的弱标签，强模型在这些标签上训练。

## 3. 从零实现

```python
def compute_pgr(weak_acc, fine_tuned_acc, ceiling_acc):
    """计算性能差距恢复（PGR）。"""
    gap = ceiling_acc - weak_acc
    if gap <= 0:
        return 0.0  # 无差距可恢复
    recovered = fine_tuned_acc - weak_acc
    return recovered / gap


# 不同弱监督质量下的 PGR
weak_accuracies = [0.50, 0.60, 0.70, 0.80]
ceiling = 0.95  # 强模型在金标签上的上限
strong_base = 0.65  # 强模型在弱标签上的直接准确率

print(f"{'弱准确率':>8}  {'PGR':>6}  {'微调准确率':>10}")
print("-" * 30)
for w_acc in weak_accuracies:
    # 强模型在弱标签上的准确率会比弱模型本身高
    # PGR 通常在 20-80% 之间
    ft_acc = w_acc + 0.5 * (ceiling - w_acc)  # 简化：50% 差距恢复
    pgr = compute_pgr(w_acc, ft_acc, ceiling)
    print(f"{w_acc:>8.1%}  {pgr:>6.1%}  {ft_acc:>10.1%}")

# 关键发现：PGR 始终为正——强模型从不完美标签中泛化
# PGR 随弱标签质量提高而增加——弱标签越好，恢复越多
```

完整代码见 `code/main.py`。

## 4. 工业工具

### 4.1 可扩展监督方法对照

| 方法 | 机制 | 假设 | 优势 |
|---|---|---|---|
| 辩论 | 两个 U 辩论，弱裁判决定 | 真实答案更容易令人信服 | 增加监督者的信息 |
| 递归奖励建模 | U 帮助训练 U+1 的 RM | RM 可递归扩展 | 监督者能力随模型增长 |
| 任务分解 | 分解为人类可检查的子任务 | 任务可分解 | 最小化人工负担 |
| W2SG | 弱标签微调强模型 | 强模型能从弱标签泛化 | 无需强监督者 |

## 5. 工程最佳实践

### 5.1 PGR 是进展代理而非解决方案

PGR 测量了从不完美监督中的泛化能力。高 PGR 不意味着对齐——可能只是任务先验填充。

### 5.2 可扩展监督和 W2SG 互补

可扩展监督增加信号质量；W2SG 从不完美信号中闭合差距。两者都需要——单独不够。

## 6. 面试考点

### Q1：PGR 是什么？为什么说它是进展代理而非解决方案？（难度：⭐⭐）

**参考答案：**
PGR = (微调后准确率 - 弱准确率) / (天花板准确率 - 弱准确率)。PGR=1.0 意味着弱监督完全闭合了差距。PGR 高说明强模型从不完美标签中泛化。但 PGR 高不意味着对齐——可能只是任务先验填充。真正的对齐需要监督者能评估模型的行为——而当模型超人时，监督者就成了瓶颈。

### Q2：三种可扩展监督机制各自的假设和优势是什么？（难度：⭐⭐⭐）

**参考答案：**
辩论：假设"找到令人信服的真实答案比虚假答案更容易"。优势：增加监督者的信息量——两个模型辩论时，弱裁判看到的是两个立场的论证。递归奖励建模：假设 U 能帮助训练 U+1 的 RM。优势：监督者的能力随模型增长。任务分解：假设任务可分解为人类可检查的子任务。优势：最小化人工负担——人类只需检查子任务。

## 📚 小结

可扩展监督和弱到强泛化是互补的。W2SG（Burns et al. 2023）测量强模型从弱标签中的泛化——PGR 在 NLP 任务上从 20% 到 80% 不等。三种可扩展监督机制：辩论（增加信息量）、递归奖励建模（监督者能力随模型增长）、任务分解（最小化人工负担）。关键局限：PGR 测量的是从不完美监督中的泛化，不是对齐——强模型可能用预训练先验填充，而非恢复真实意图。
