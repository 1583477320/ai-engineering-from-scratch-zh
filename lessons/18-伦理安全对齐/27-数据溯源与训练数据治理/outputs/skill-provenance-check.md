# 数据溯源检查技能

## 目标
给定用于训练的数据集，检查AB 2013的12字段覆盖、退出基础设施合规、DPA一致性和不可逆性风险评估。

## 检查流程

### 1. AB 2013字段覆盖检查

**12字段验证：**
- [ ] 字段1：来源或所有者
- [ ] 字段2：如何促进预期目的
- [ ] 字段3：数据点数量
- [ ] 字段4：数据点类型
- [ ] 字段5：是否包含受保护内容
- [ ] 字段6：是否购买或许可
- [ ] 字段7：是否包含个人信息
- [ ] 字段8：是否包含汇总消费者信息
- [ ] 字段9：清理处理描述
- [ ] 字段10：收集时间段
- [ ] 字段11：首次使用日期
- [ ] 字段12：是否使用合成数据

### 2. 退出基础设施合规

**机器可读退出信号：**
- robots.txt
- C2PA"无AI训练"声明
- TDM.Reservation

**检测方法：**
- 解析robots.txt
- 检查C2PA元数据
- 验证TDM信号

### 3. DPA一致性评估

**2025年立场总结：**
- 爱尔兰DPC：合法利益 + 保障措施
- UK ICO：积极响应 + 持续监控
- 汉堡DPA：为EU一致性放弃紧急程序
- 科隆法院：退出即足够

**评估清单：**
- 是否有退出机制？
- 是否有保障措施？
- 是否有透明性？
- 是否有监控？

### 4. 不可逆性风险评估

**风险因素：**
- 数据是否包含个人信息？
- 数据是否被广泛使用？
- 模型是否已部署？

**缓解措施：**
- 收集时合规
- 实现退出机制
- 记录数据使用
- 准备重新训练计划
