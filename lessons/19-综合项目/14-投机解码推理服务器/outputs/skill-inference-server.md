# 推理服务器技能

## 目标
以2.5x+基线吞吐量服务两个开源模型，提供完整尾延迟报告。

## 评估标准

| 权重 | 标准 | 测量方法 |
|:---:|------|----------|
| 25 | 实测加速比 | 两个模型上匹配质量的2.5x+吞吐量 |
| 20 | 接受率 | 按分布划分的接受率报告 |
| 20 | p99尾延迟 | 有/无投机时不同批大小的p99 |
| 20 | 运维 | K8s部署、HPA队列等待、平稳上线 |
| 15 | 报告和方法论 | 清晰解释变更和原因 |

## 构建检查清单

- [ ] 目标模型准备（FP8量化）
- [ ] EAGLE-3草案头加载
- [ ] 基线吞吐量测量
- [ ] 启用EAGLE-3后对比
- [ ] 接受率按流量分布报告
- [ ] p50/p99延迟报告（多个批大小）
- [ ] K8s HPA部署
- [ ] 成本对比$/100万token
