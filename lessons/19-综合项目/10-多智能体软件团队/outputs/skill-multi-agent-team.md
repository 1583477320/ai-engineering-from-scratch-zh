# 多智能体软件团队技能

## 目标
给定问题URL和并行度，团队产出可合并的PR，含每角色token会计。

## 评估标准

| 权重 | 标准 | 测量方法 |
|:---:|------|----------|
| 25 | SWE-bench Pro pass@1 | 50问题子集的pass@1 |
| 20 | 并行加速比 | 墙钟时间 vs 单智能体基线 |
| 20 | 评审质量 | 注入bug探测的假批准率 |
| 20 | Token效率 | 每解决问题总token vs 单智能体 |
| 15 | 协调工程 | 合并冲突解决、交接失败直方图 |

## 构建检查清单

- [ ] 类型化消息任务板
- [ ] 架构师计划+子任务分解
- [ ] N个并行编码员（git worktree + Daytona）
- [ ] 合并协调器（三方合并）
- [ ] 评审者把关（不允许自批准）
- [ ] 测试者隔离验证
- [ ] 交接会计（token/角色）
- [ ] 50问题的SWE-bench Pro评测
