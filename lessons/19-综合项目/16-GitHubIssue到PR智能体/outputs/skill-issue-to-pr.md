# GitHub Issue到PR智能体技能

## 目标
构建GitHub App+异步云工作器，将标记的issue转化为可审阅的PR，带有限预算和作用域凭证。

## 评估标准

| 权重 | 标准 | 测量方法 |
|:---:|------|----------|
| 25 | 30个issue的通过率 | CI绿+覆盖率OK的端到端成功率 |
| 20 | PR质量 | diff大小、覆盖率增量、风格合规 |
| 20 | 每解决issue的成本和延迟 | 每PR的$和墙钟时间 |
| 20 | 安全性 | 作用域令牌、每仓库预算、不强制推送、凭证卫生 |
| 15 | 操作者UX | 理据评论、重试能力、@提及后续 |

## 构建检查清单

- [ ] GitHub App（细粒度令牌）
- [ ] Webhook接收器+队列
- [ ] 分发器（预算执行）
- [ ] 环境推断（自动Dockerfile）
- [ ] 智能体循环（预算限制）
- [ ] 沙箱内CI验证
- [ ] PR发布（理据+diff+trace）
- [ ] 30个issue评测
