# InternVL3：原生多模态预训练

> InternVL3 之前的每个开源 VLM 都遵循相同的三步配方：拿一个在数万亿文本词元上训练的文本 LLM，接上视觉编码器，然后微调衔接。这有效但有对齐债务——文本 LLM 将全部预训练预算花在了纯文本上，不原生理解视觉词元。InternVL3（2025 年 4 月）拒绝了事后方案：一次预训练，文本和多模态从第一步就交错。结果在 78B 参数开源模型上匹配 Gemini 2.5 Pro 的 MMMU-Pro。

## 1. 问题

2024 年的 VLM 都是"事后拼接"：先训练一个纯文本 LLM，再加一个视觉编码器，然后微调。问题是：文本 LLM 的全部预训练预算都花在了纯文本上——它不"理解"视觉词元。加视觉时，LLM 需要重新学习如何关联视觉输入和文本推理——同时不能忘记文本能力。这叫**对齐债务**。

InternVL3 的答案：**从第一步就用混合文本+多模态数据预训练**——LLM 同时学会文本和视觉理解。

---

## 2. 概念

### 2.1 对齐债务的三个症状

| 症状 | 描述 | 解决方案 |
|------|------|---------|
| **灾难性遗忘** | 视觉微调后文本能力退化 | 保持冻结 / 混合训练 |
| **回答漂移** | 模型开始编造视觉内容 | 从零训练 |
| **视觉-文本不一致** | 图像和文本理解不对齐 | 原生多模态预训练 |

### 2.2 事后方案 vs 原生方案

| 方面 | 事后方案（LLaVA） | 原生方案（InternVL3） |
|------|-----------------|---------------------|
| 预训练数据 | 纯文本 | 文本+多模态混合 |
| 视觉编码器 | 后加的 | 同时训练 |
| 对齐债务 | 有 | 无 |
| 训练成本 | 低 | 高（但只需一次） |
| 最终质量 | 接近原生 | 略好 |

### 2.3 InternVL3 的训练配方

```
阶段 1: 文本预训练（与标准 LLM 相同）
阶段 2: 视觉-文本对齐（CLIP 风格对比学习）
阶段 3: 多模态预训练（文本+图像交错）
阶段 4: 视觉指令微调（特定任务）
```

**关键区别**：阶段 3 是"原生多模态"——从第一步就用混合数据。

### 2.4 架构设计

| 组件 | InternVL3 选择 | 原因 |
|------|---------------|------|
| 视觉编码器 | InternViT-6B | 原生多模态 |
| 连接器 | MLP 投影器 | 简单有效 |
| LLM | InternLM2 | 原生多模态预训练 |

## 6. 工程最佳实践

### 6.1 原生预训练的数据比例

- 阶段 1（纯文本）：80% 文本 + 20% 多模态
- 阶段 3（原生多模态）：50% 文本 + 30% 图像 + 20% 视频
- 数据比例直接影响模型在各任务上的能力分布

### 6.2 踩坑经验

- **训练成本高**：原生预训练比事后微调贵 5-10 倍
- **需要大规模数据**：纯文本和多模态数据都需要万亿级规模
- **灾难性遗忘**：即使原生预训练，过度微调仍可能退化文本能力

## 7. 常见错误

### 错误 1：只用多模态数据预训练

**现象：** 模型文本能力退化——无法生成连贯的文本回答。

**原因：** 多模态数据中文本部分不够丰富——LLM 需要大量纯文本学习语言结构。

### 错误 2：阶段 3 训练不充分

**现象：** 视觉和文本理解不对齐——图像回答与文本逻辑不一致。

**原因：** 原生多模态预训练需要足够长——通常需要数千步。

## 8. 面试考点

### Q1：为什么原生多模态预训练比事后微调更好？（难度：⭐⭐）

**参考答案：**
事后微调（如 LLaVA）先在纯文本上训练 LLM，再加视觉编码器。这意味着 LLM 的全部预训练预算都花在了文本上——它不"理解"视觉词元。加视觉时，LLM 需要重新学习如何关联视觉输入和文本推理，同时不能忘记文本能力（灾难性遗忘）。原生预训练从第一步就用混合数据——LLM 同时学习文本和视觉理解，没有对齐债务。

### Q2：对齐债务的三个症状是什么？（难度：⭐⭐⭐）

**参考答案：**
(1) **灾难性遗忘**：视觉微调后 LLM 的文本生成质量下降——因为视觉数据更新了 LLM 权重；(2) **回答漂移**：模型开始编造视觉内容（如描述图像中不存在的物体）——因为 LLM 的文本能力没有与视觉输入对齐；(3) **视觉-文本不一致**：图像和文本理解不对齐——模型可能正确识别图像内容但无法用文本准确描述。

## 📚 小结

InternVL3 拒绝了事后方案——从第一步就用混合文本+多模态数据预训练。对齐债务的三个症状：灾难性遗忘、回答漂移、视觉-文本不一致。原生多模态预训练虽然更贵但避免了这些问题。78B 参数在 MMMU-Pro 上匹配 Gemini 2.5 Pro——证明了原生方案的有效性。
