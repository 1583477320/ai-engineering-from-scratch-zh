# 文档与图表理解

> 文档不仅包含文本——表格、图表、公式、布局都承载着关键信息。OCR 提取文本，但丢失了空间关系；表格识别器提取结构，但丢失了语义。多模态 VLM 可以同时理解文档的所有模态——文本、图像、表格、布局——实现真正的文档智能。

## 1. 问题

一份公司年度报告包含：文本段落、收入表格、增长趋势图表、财务公式。传统方法需要三个独立的模型——OCR 提取文本、表格识别器提取表格、图表分析器提取图表信息。信息在三个模型之间传递时损失严重。

**多模态 VLM 的答案：直接让模型"看"整个页面——同时理解文本、表格和图表。**

## 2. 概念

### 2.1 文档理解的三种模态

| 模态 | 描述 | 传统方法 | VLM 方法 |
|------|------|---------|---------|
| **文本** | 段落、标题、列表 | OCR | 直接嵌入 |
| **表格** | 结构化行列数据 | 表格识别器 | 直接嵌入 |
| **图表** | 可视化数据 | 图表分析器 | 直接嵌入 |

### 2.2 VLM 文档理解

```
文档页面 → [VLM 编码器] → 页面特征
                ↓
         [LLM 推理] → 回答文档相关问题
```

VLM 的优势：**不需要分割文档的各个部分——直接理解整个页面。**

## 3. 从零实现

### Step 1：表格提取（简化版）

```python
def extract_table_structure(image, detector):
    """从图像中提取表格结构。"""
    # 简化版：假设已有表格检测结果
    tables = detector.detect(image)
    return [{"bbox": t["bbox"], "rows": t["rows"], "cols": t["cols"]} for t in tables]
```

### Step 2：图表数据提取

```python
def extract_chart_data(chart_image, parser):
    """从图表中提取数据点。"""
    data_points = parser.parse(chart_image)
    return {
        "type": chart_image.get("chart_type", "unknown"),
        "x_label": chart_image.get("x_label", ""),
        "y_label": chart_image.get("y_label", ""),
        "data_points": data_points,
    }
```

### Step 3：VLM 文档问答

```python
def document_qa(vlm, document_image, question):
    """使用 VLM 进行文档问答。"""
    prompt = f"根据以下文档页面，回答问题：{question}"
    response = vlm.generate(document_image, prompt)
    return response
```

## 6. 工程最佳实践

### 6.1 文档处理管道

```
文档输入 → [文档分类] → [OCR] → [表格/图表识别] → [VLM 问答] → 结果
```

### 6.2 踩坑经验

- **多栏布局混乱**：VLM 可能错误理解列关系——使用版面分析预处理
- **表格跨页断裂**：长表格被分到两页——需要跨页关联
- **低分辨率图像**：扫描件质量差——使用超分辨率预处理

---

## 7. 常见错误

### 错误 1：忽略文档布局信息

**现象：** VLM 正确识别了每个元素，但错误理解它们的空间关系。

**修复：** 使用布局分析模型（如 LayoutLM）预处理文档结构。

### 错误 2：对纯文本文档使用视觉 VLM

**现象：** 检索速度慢、成本高。

**修复：** 纯文本文档使用文本 RAG；只对图表丰富的文档使用视觉 RAG。

## 8. 面试考点

### Q1：VLM 文档理解和传统 OCR+NLP 管道有什么区别？（难度：⭐⭐）

**参考答案：**
传统方法将文档拆分为三个独立模块：OCR（文字）+ 表格识别器（结构）+ 图表分析器（数据）。信息在模块间传递时损失——OCR 丢弃图表数据，表格识别器丢弃语义。VLM 直接"看"整个页面——同时理解文本、表格、图表和布局，不需要分割。VLM 的优势是全局理解，劣势是计算成本更高。

## 📚 小结

文档理解需要同时处理文本、表格、图表和布局。VLM 直接"看"整个页面——不需要分割。ColPali 实现了视觉原生文档检索——在图表丰富的文档上比文本 RAG 高 20-40%。关键基准：DocVQA、ChartQA、MMMU。
