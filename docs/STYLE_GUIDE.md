# 风格指南 (Style Guide)

> **规则：** 所有文档必须遵循本指南。风格一致性直接影响读者的学习体验和信任感。

---

## 1. 写作基调

### 1.1 像老师，不像博主

本项目的定位是**教材**，不是技术博客。区别在于：

| | 博客风格（❌） | 教材风格（✓） |
|---|---|---|
| 开篇 | "最近 AI 很火，今天我们来聊聊…" | "你的模型不认识中文。它只认识数字。" |
| 论证 | "据说Transformer效果很好" | "Transformer 用自注意力替代了循环结构。以下是具体原因。" |
| 结尾 | "大家觉得呢？欢迎留言讨论" | "下一课我们将把多个注意力头组合在一起，构建多头注意力。" |
| 引用 | "有人说…" | "Vaswani et al. (2017) 在论文中证明…" |

**核心原则：**
- 自信而不傲慢 — 你确定你写的东西是对的
- 直接而不粗鲁 — 不绕弯子，但保持尊重
- 具体而不模糊 — 写"上下文窗口从 128K 缩水到 30K"，不写"性能变得更差"

### 1.2 有态度

每课应该有一个核心观点，贯穿始终。读者读完应该能说出"这节课的核心洞察是什么"。

```markdown
# ❌ 中性写法
本章将介绍 Transformer 架构中的自注意力机制。

# ✓ 有态度的写法
自注意力不是一个"更好的 RNN"。它是一个完全不同的东西 —— 一个软性的数据库查询，让每个词元能直接与整个序列对话。
```

### 1.3 面向读者

假设读者：
- 有编程经验（Python），但不一定有 ML 背景
- 聪明但时间有限 — 需要知道"为什么学这个"和"实际怎么用"
- 想深入理解原理，而不是停留在调包

不要假设读者已经看过原始论文。每次引入新概念时，先给直觉，再给定义。

## 2. 文档结构

### 2.1 标题层级

```markdown
# 一级标题 — 仅用于课程标题（每课一个）
## 二级标题 — 用于章节标题（如 ## 1. 问题）
### 三级标题 — 用于子章节（如 ### 1.1 直观理解）
#### 四级标题 — 尽量避免，如需使用则为子子章节
```

**规则：**
- 一级标题（`#`）：每课最多一个
- 二级标题（`##`）：章节级，序号 + 中文标题，如 `## 1. 问题`
- 三级标题（`###`）：子章节级，如 `### 2.1 直观理解`
- 四级标题（`####`）：尽量不用。如果内容需要这么深的嵌套，考虑拆成多个子章节
- 不允许跳级（如 `##` 下面直接出现 `####`）

### 2.2 章节编号

章节使用数字编号（1、2、3…），子章节使用 `数字.数字` 格式（2.1、2.2…）。

```markdown
## 1. 问题
### 1.1 为什么需要这个
### 1.2 不做会怎样

## 2. 概念
### 2.1 直观理解
### 2.2 形式化定义
```

特殊章节（学习目标、关键术语、小结、练习、产出、参考资料）使用 emoji 标记：

```markdown
## 🎯 学习目标
## 🔑 关键术语
## 📚 小结
## ✏️ 练习
## 🚀 产出
## 📖 参考资料
```

## 3. 中文写作规范

### 3.1 中英文混排

**中文与英文之间加空格，中文与数字之间加空格。**

```markdown
# ❌ 错误
使用Python 3.11训练一个BERT模型

# ✓ 正确
使用 Python 3.11 训练一个 BERT 模型
```

**例外：** 中文标点符号与英文/数字之间不加空格。

```markdown
# ✓ 正确
大语言模型（如 GPT-4）支持 128K 上下文窗口。

# ❌ 错误
大语言模型 （ 如 GPT-4 ） 支持 128K 上下文窗口 。
```

### 3.2 标点符号

正文中使用中文全角标点，代码和公式中使用英文半角标点。

| 场景 | 使用 |
|---|---|
| 正文中的逗号、句号 | `，` `。`（全角） |
| 正文中的括号 | `（）`（全角） |
| 正文中的引号 | `""`（全角） |
| 代码中的一切 | `,` `.` `()` `""`（半角） |
| 数学公式 | 按 LaTeX 规范 |

```markdown
# ✓ 正确
自注意力机制（Self-Attention）是 Transformer 的核心。它的计算公式如下：

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

其中 $d_k$ 是键向量的维度，"缩放"的目的是防止梯度消失。
```

### 3.3 专有名词

| 类型 | 规则 | 示例 |
|---|---|---|
| 人名 | 首次出现用"中文译名（英文原名）"，后续直接写中文 | 瓦斯瓦尼（Vaswani），后续直接写 Vaswani |
| 公司名 | 保留英文 | OpenAI、Meta、Google |
| 产品名 | 保留英文 | ChatGPT、Claude、GitHub Copilot |
| 模型名 | 保留英文 | GPT-4、Llama 3、BERT |
| 框架/库名 | 保留英文 | PyTorch、LangChain、vLLM |
| 论文名 | 保留英文原文 | "Attention Is All You Need" |
| 技术术语 | 按术语表 | 见 [`TERMINOLOGY.md`](./TERMINOLOGY.md) |

### 3.4 不使用口语化表达

```markdown
# ❌ 口语化
这个操作巨慢，基本上没法用。你想想看，一个 70 亿参数的模型...

# ✓ 书面化
这个操作的计算开销很大。对于一个 70 亿参数的模型...

# ❌ 口语化
把数据丢进去就行了

# ✓ 书面化
将数据输入模型即可
```

## 4. Markdown 格式

### 4.1 段落

- 一个段落不超过 5 行。超过则拆开。
- 段落之间空一行。
- 不要连续使用多个空行（最多 1 个）。

### 4.2 强调

- **加粗**用于关键概念首次出现时，或需要读者特别注意的内容
- *斜体*尽量少用—中文排版中斜体可读性差
- `行内代码`用于：变量名、函数名、文件名、命令行参数、短代码片段
- 不要用加粗 + 斜体叠加

```markdown
# ✓ 正确
**自注意力（Self-Attention）** 是 Transformer 的核心机制。

调用 `scaled_dot_product_attention(Q, K, V)` 即可完成一次注意力计算。

# ❌ 错误
***自注意力***非常非常重要！！
```

### 4.3 列表

```markdown
# ✓ 无序列表（并列关系）
- PyTorch
- Transformers
- Datasets

# ✓ 有序列表（有先后顺序）
1. 安装依赖
2. 加载数据
3. 训练模型

# ❌ 错误：混用符号
- 第一步
* 第二步
- 第三步
```

- 列表项超过一行时，后续行缩进对齐
- 列表项之间不空行（除非某项内部有代码块或多段落）

### 4.4 代码块

**始终标注语言。**

```markdown
# ✓ 正确
\`\`\`python
import torch
x = torch.tensor([1, 2, 3])
\`\`\`

# ❌ 错误（未标注语言）
\`\`\`
import torch
\`\`\`
```

- Python 代码块优先
- bash 命令标注为 `bash`
- 输出/日志标注为 `text` 或 `plaintext`
- 代码和输出分开两个块，不要混在一个块里

### 4.5 链接

```markdown
# ✓ 使用描述性链接文本
详见 [PyTorch MultiheadAttention 文档](https://pytorch.org/docs/...)

# ❌ 裸露 URL
详见 https://pytorch.org/docs/...

# ❌ 无意义的链接文本
点击[这里](url)查看
```

### 4.6 表格

保持对齐，表头和分隔符必写。

```markdown
| 模型 | 词表大小 | 分词器类型 |
|---|---|---|
| BERT | 30,522 | WordPiece |
| GPT-2 | 50,257 | Byte-level BPE |
| GPT-4 | ~100,256 | Byte-level BPE |
```

## 8. 数学公式

使用 LaTeX 语法，行内和行间公式采用标准写法。

```markdown
# 行内公式
缩放因子为 $\sqrt{d_k}$，其中 $d_k$ 是键向量的维度。

# 行间公式
$$
\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right) V
$$

# 多行公式（对齐）
$$
\begin{aligned}
\text{Scores} &= QK^T \\
\text{Weights} &= \text{softmax}\left(\frac{\text{Scores}}{\sqrt{d_k}}\right) \\
\text{Output} &= \text{Weights} \cdot V
\end{aligned}
$$
```

- 公式是句子的一部分时，用行内公式
- 公式需要独立展示时，用行间公式
- 每个符号首次出现时需要解释

## 9. 常见错误速查

| ❌ 错误 | ✓ 正确 | 说明 |
|---|---|---|
| 中文English之间无空格 | 中文 English 之间加空格 | 中英文混排规则 |
| Use English in Chinese docs | 在中文文档中使用中文 | 不要中英夹杂 |
| `token` → Token | `词元` → 词元 | 有标准翻译的术语用中文 |
| ~~删除线表示改动~~ | 直接写成最终版本 | 本项目不是翻译项目 |
| 使用 HTML 标签 | 使用 Markdown 语法 | `<img>` → `![]()` |
| `pip install torch==1.13` | `pip install torch>=2.0` | 使用现代版本号 |
| 这个很简单 / 显然 / 众所周知 | （直接说明内容） | 不预设读者已经知道 |

## 10. 自检清单

写完一篇文档后，检查：

- [ ] 中英文之间有空格
- [ ] 中文使用全角标点，代码使用半角标点
- [ ] 写作基调符合教材风格（不口语化、不博客化）
- [ ] 从痛点出发，不是从定义出发
- [ ] 首次出现的术语对照术语表
- [ ] 代码块标注了语言
- [ ] 表格有表头分隔线
- [ ] 链接使用描述性文本，无裸露 URL
- [ ] 公式中的符号有解释
- [ ] 没有"显然"、"很简单"、"众所周知"等预设
- [ ] 标题层级不跳级
- [ ] 文件名符合命名规范（小写英文 + 连字符）
